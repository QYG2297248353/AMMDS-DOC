# 站点认证

本文介绍 AMMDS 的站点认证功能，认证通过后可使用榜单、检索等高级功能。

{/* truncate */}

站点认证就是让系统跟某个网站"对上号"。认证通过后，就能用榜单、检索这些高级功能了。

## 访问站点认证页面

通过以下路径进入：

**后台右上角头像下拉菜单 >> 站点认证**

## 认证方法

### 选择认证站点

从下拉列表里选一个你有账号的站点。

### 填写认证信息

填一下这几样：

- **UID**：你在该站点的用户标识，一般是数字。有些站点用的是注册时填的用户名
- **PassKey**：站点给你的密钥，相当于你的站点"密码"

### 提交认证

填好后点提交就完事了。

## UID 与 PassKey 获取方法

### UID 获取

- 大多数站点在个人资料页能看到 UID
- 有些站点在设置或账户信息里
- 少数站点的 UID 就是你注册时用的用户名

### PassKey 获取

不同站点的获取方式不一样，一般可以从这些地方找到：

- **站点设置**：在用户设置或安全设置里找 API 密钥或 PassKey 选项
- **个人资料**：有些站点在个人资料页提供查看或重置 PassKey 的功能
- **账户安全**：在跟安全相关的页面找密钥管理选项

## 注意事项

:::warning[重要提示]
- **别把 PassKey 泄露了**：PassKey 就相当于你的站点密码，别告诉别人
- **认证信息要填对**：确保 UID 和 PassKey 填正确，不然认证失败
- **定期更新**：如果你在站点改了密码等信息，可能需要重新认证
- **遵守站点规则**：用认证功能时，要遵守各站点的使用规定
:::

## 常见问题

### Q: 站点认证有啥用？

**A**: 认证后，你就可以用系统的榜单、检索等高级功能了，体验更完整。

### Q: 认证失败怎么办？

**A**: 可能的原因：
- UID 或 PassKey 填错了
- 站点服务器暂时连不上
- 你这边网络有问题

检查一下认证信息，确认站点能正常访问，再试一次。

### Q: 可以同时认证多个站点吗？

**A**: 可以，你想认证几个就认证几个，每个站点的信息是独立的。

### Q: PassKey 泄露了怎么办？

**A**: 马上去对应站点重置 PassKey，然后在系统里更新认证信息。

## 最佳实践

- **定期检查认证状态**：确保认证一直有效
- **在安全的网络环境下认证**：别在公共 WiFi 下搞认证
- **保管好认证信息**：建议记下来，以后要用
- **遵守站点规定**：不要违规使用
- **信息变了及时更新**：如果在站点改了信息，记得同步更新到系统里
