# 刮削配置

本文介绍 AMMDS 的刮削配置，包括元数据获取方式、文件整理格式和字符过滤等设置项说明。

{/* truncate */}

刮削配置用来告诉系统：从网上找电影信息的时候怎么找（比如封面、简介、演员），整理好之后文件按什么格式存放。

> **啥是"刮削"？** 刮削就是让系统自动去网上搜索电影的信息，比如标题、封面海报、演员表、简介这些（统称"元数据"），然后下载下来配上你的视频文件。

## 访问刮削配置

通过下面的路径就能找到刮削规则配置页面：

**任务管理 >> 刮削规则配置**

:::info[操作提示]
改完配置记得点 **保存**，不然不会生效。
:::

## 基础配置

### 字符过滤

**干啥的**：刮削前先把文件名里的乱七八糟字符过滤掉，提高识别准确率。

**有啥用**：去掉文件名里的干扰字符，系统就能更准地识别出影片编号，从而找到更准确的影片信息。

### 允许图片下载失败

**干啥的**：就算图片下载失败了，刮削任务也继续往下走，不卡住。

**有啥用**：开了这个选项，个别图片下不来也不影响刮削结果，整体成功率更高。

### 整理目录格式

**干啥的**：设置刮削完成后，文件夹怎么组织。

**怎么配**：用系统提供的变量组合出你想要的目录格式。

**示例**：`${actor}/${number}` 会按"演员名/影片编号"这样两层目录来放文件。

### 整理文件名格式

**干啥的**：设置刮削完成后，视频文件怎么命名。

**怎么配**：用系统提供的变量组合出你想要的文件名格式。

**示例**：`${title_safe}.${ext}` 会用安全的影片标题做文件名，保留原来的扩展名。

### 自动刮削

**干啥的**：扫描任务完成后，要不要自动开始刮削。

**默认行为**：默认要手动选文件再去刮削。

**开启效果**：开了之后，系统会自动找到可以刮削的电影文件并开始干活。

### 自动刮削间隔

**干啥的**：自动刮削多久跑一次。

**单位**：分钟

**默认值**：3分钟（开启自动刮削的情况下）

### 高清封面下载

**干啥的**：要不要下载影片的高清封面图。

**下载来源**：开了会先尝试从 Fanza DMM 下高清封面，没有的话再从其他地方找。

### 外挂字幕搜索

**干啥的**：要不要搜刮并下载影片的外挂字幕（软字幕）。

**存储位置**：下载到的字幕文件会保存到最终的刮削目录里。

### 番号混合

**干啥的**：刮削后生成的信息文件（nfo）里，标题等字段要不要加上番号前缀。

### 自动标签

**干啥的**：刮削后生成的信息文件（nfo）里，要不要在 tag 标签里加额外信息。

**示例**：比如加上分辨率、番号、中字、破解之类的标签。

### 自动分类标签

**干啥的**：刮削后生成的信息文件（nfo）里，要不要在 genre 标签里加额外信息。

**示例**：比如加上分辨率、番号、中字、破解之类的标签。

### 本地数据优先

**干啥的**：刮削时优先用本地已有的数据，不上网去搜。

**有啥用**：如果本地数据已经符合要求，就直接用本地的，不去下载了，刮削更快。

### 刮削器

**干啥的**：选择用哪个信息源去搜索影片信息。

**可选的有**：
- **本地**：用你自己手动创建的影视信息
- **Metatube** (插件)
- **ThePornDB** (插件)
- **Stash** (插件)
- **Fanza DMM** (插件)

### 刮削剧照

**干啥的**：要不要下载影片的剧照（影片截图、场景图）。

**存储位置**：下载的剧照会保存到你指定的目录里。

### 剧照目录

**干啥的**：设置剧照文件存放在哪个文件夹。

**默认值**：默认放在视频文件同目录下的 `extrafanart` 文件夹里。

### 多线程执行

**干啥的**：要不要同时开多个线程来刮削（同时处理多个文件，更快）。

**优缺点**：
- **优点**：刮削更快
- **缺点**：更吃系统资源

**建议**：根据你电脑配置来。比如 CPU 有 4 个核心，建议设 4 个线程。

### 最大线程数

**干啥的**：刮削时最多同时开多少个线程。

**默认值**：5 个线程

**建议**：根据你电脑配置来，别设太高把系统搞卡了。

## 目录与文件名整理

配置里有 **整理目录格式** 和 **整理文件名格式** 两个选项。你可以组合系统提供的变量，自定义目录和文件名的组织方式。

### 安全变量

**干啥的**：在变量名后加 `_safe` 后缀，可以避免文件名里出现特殊符号导致的问题。

**示例**：`${actor_safe}` 如果演员名里有特殊字符，会被替换成安全的字符。

**有啥用**：避免影片标题里的特殊符号把目录搞乱。

### 整理示例

一个完整的视频文件整理后大概是这样的：`${actor}/${number}/${title_safe}.${ext}`

**举个例子**：
- 演员名："John Doe"
- 影片编号："ABC-123"
- 影片标题："Sample Title"
- 文件扩展名："mkv"

**最终路径**：`John Doe/ABC-123/Sample Title.mkv`

### 整理目录变量

**默认配置**：`${actor}/${number}`

**注意**：`/` 会被当成分隔符，可以用来创建不同的目录层级。

| 变量名 | 安全变量名 | 说明 | 示例 |
|--------|------------|------|------|
| `${actor}` | `${actor_safe}` | 演员名称 | "John Smith" |
| `${number}` | `${number_safe}` | 影片编号 | "ABC-123" |
| `${title}` | `${title_safe}` | 影片标题 | "Sample Title" |
| `${date}` | `${date_safe}` | 发行日期 | "2024/01/01" |
| `${year}` | `${year_safe}` | 发行年份 | "2024" |
| `${month}` | `${month_safe}` | 发行月份 | "01" |
| `${day}` | `${day_safe}` | 发行日期 | "01" |
| `${mosaic}` | `${mosaic_safe}` | 马赛克类型 | "无码/有码" |
| `${resolution}` | `${resolution_safe}` | 视频分辨率 | "1080p" |
| `${director}` | `${director_safe}` | 导演名称 | "Director Name" |
| `${studio}` | `${studio_safe}` | 制作商名称 | "Studio Name" |

**默认值**：所有变量都有默认值，没指定的话就用默认的。

### 整理文件名变量

**默认配置**：`${number}`

| 变量名 | 安全变量名 | 说明 | 示例 |

## 常见问题

### Q: 为什么刮削失败？

**A**：可能的原因有：
- 番号识别错了
- 网络连不上
- 刮削器找不到匹配的信息
- 配置配错了

### Q: 怎么提高刮削成功率？

**A**：可以试试这些方法：
- 确保文件名里的番号清晰可辨
- 开启字符过滤功能
- 配置多个刮削器当备用
- 确保网络连接正常

### Q: 自动刮削和手动刮削有啥区别？

**A**：
- **自动刮削**：扫描完自动开始，不用你管，适合批量处理
- **手动刮削**：你得自己选文件再刮，适合对结果要求比较精细的情况

### Q: 为啥下载的字幕没跟视频对上？

**A**：可能的原因有：
- 字幕搜索时番号识别不准
- 网络问题导致字幕没下下来
- 字幕文件命名跟视频文件名对不上

### Q: 怎么自定义目录结构？

**A**：组合系统提供的变量就行，比如：
- `${studio}/${actor}/${number}`：按"制作商 → 演员 → 番号"来组织
- `${year}/${month}/${title_safe}`：按"年份 → 月份 → 标题"来组织
