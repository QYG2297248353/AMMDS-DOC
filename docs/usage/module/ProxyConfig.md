# 网络代理配置

网络代理配置用来设置程序上网时走哪个代理服务器。有些网络资源需要"翻墙"才能访问，代理就是干这个的。

<!-- truncate -->

## 访问网络代理配置

可以通过下面两个路径进入配置页面：

1. **任务管理 >> 网络代理**
2. **系统管理 >> 系统配置 >> 网络配置**

## 配置项说明

### 开启代理

**干啥的**：要不要开代理功能。

**有啥用**：开了之后，程序会通过你配置的代理服务器去访问网络。

### 代理协议

**干啥的**：选代理的类型。

**可选类型**：
- HTTP
- SOCKS4
- SOCKS5

**建议**：根据你的代理服务商提供的类型来选。

### 代理主机

**干啥的**：填代理服务器的地址。

**格式**：可以是 IP 地址（比如 `192.168.1.1`）或者域名（比如 `proxy.example.com`）。

### 代理端口

**干啥的**：填代理服务器的端口号。

**常见端口**：
- HTTP 代理：8080、3128
- SOCKS 代理：1080

### 用户名

**干啥的**：如果代理服务器需要登录认证，填用户名。

### 密码

**干啥的**：如果代理服务器需要登录认证，填密码。

### 免代理主机

**干啥的**：设置哪些地址不用走代理，直接访问。

**添加方法**：输入地址后按回车确认。

**适用场景**：
- 访问内网资源（比如局域网里的 NAS）
- 不需要代理也能上的网站

## 配置示例

### 示例 1：HTTP 代理

| 配置项 | 设置值 |
|--------|--------|
| 开启代理 | 开启 |
| 代理协议 | HTTP |
| 代理主机 | 192.168.1.100 |
| 代理端口 | 8080 |
| 用户名 | user |
| 密码 | password |
| 免代理主机 | localhost, 127.0.0.1 |

### 示例 2：SOCKS5 代理

| 配置项 | 设置值 |
|--------|--------|
| 开启代理 | 开启 |
| 代理协议 | SOCKS5 |
| 代理主机 | proxy.example.com |
| 代理端口 | 1080 |
| 用户名 | - |
| 密码 | - |
| 免代理主机 | localhost, 192.168.1.* |

## 注意事项

:::warning 重要提示
- **配置完要重启程序才能生效**
- **配错了程序可能跑不起来**
- **确保代理地址和端口填对了**
- **确保代理服务器能正常用**
- **需要认证的代理，确保用户名密码正确**
:::

## 常见问题

### Q: 为啥要配代理？

**A**：有些网络资源需要特定网络环境才能访问。比如：
- 某些刮削器需要访问特定地区的资源
- 某些插件要调用外部 API

### Q: 怎么测试代理配没配对？

**A**：可以这样试：
1. 配好代理
2. 重启程序
3. 试一下需要联网的操作（比如刮削）
4. 看能不能成功

### Q: 开了代理后，所有请求都走代理吗？

**A**：不是，只有需要联网的特定操作会走代理。而且你还可以设置"免代理主机"，指定哪些不走代理。

### Q: 设置了免代理主机，但还是走了代理，为啥？

**A**：可能的原因有：
- 免代理主机的格式写错了
- 配置还没生效（需要重启）
- 程序内部逻辑优先走代理

### Q: 代理连不上会怎样？

**A**：可能导致以下问题：
- 刮削失败
- 插件用不了
- 拿不到在线资源

## 最佳实践

- **用稳定的代理**：别用老掉线的代理，不然动不动就失败
- **合理设置免代理**：本地资源和直连能访问的地址加到免代理列表，速度更快
- **定期检查**：网络环境变了，及时更新代理配置
- **备份配置**：改之前记下当前配置，万一出问题能恢复
