# Jellyfin

Jellyfin 是一套开源的多媒体管理工具，专门用来整理、管理和分享你的数字媒体文件。通过这个插件，AMMDS 可以和 Jellyfin 媒体服务器深度集成，实现元数据同步、演员头像自动补全等功能。

## 插件配置

![插件配置](/img/plugin/jellyfin-01.png)

### 基本配置

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 启用状态 | 控制是否启用 Jellyfin 插件 | 关闭 |
| 服务地址 | Jellyfin 服务器的访问地址 | `http://127.0.0.1:8096` |
| API 密钥 | Jellyfin 服务器的认证令牌 | - |
| 用户名 | (可选) Jellyfin 服务器的登录用户名 | - |
| 密码 | (可选) Jellyfin 服务器的登录密码 | - |
| 绑定用户 | 选择要绑定的 Jellyfin 用户账号 | - |
| 自动补全 | 控制是否自动给媒体库里缺失头像的演员补上头像 | 关闭 |

![插件配置](/img/plugin/jellyfin-02.png)

### 配置说明

- **服务地址**：请填服务器的实际网络地址，别用 `localhost` 或 `127.0.0.1`，否则用 Docker 部署时可能连不上
- **API 密钥**：在 Jellyfin 管理界面「系统」→「高级」→「API 密钥」里生成
- **绑定用户**：要先填好服务地址和 API 密钥并保存，才能在下拉列表里选用户
- **自动补全**：开启后，系统会自动给媒体库中缺失头像的演员补上头像，前提是 AMMDS 里已经存了对应演员的头像

### 连接验证

- **网络连通性**：确保 AMMDS 服务器能访问 Jellyfin 的服务地址
- **认证信息**：API 密钥需要有足够的权限来执行操作
- **服务状态**：确保 Jellyfin 服务器在正常运行，能正常响应 API 请求

## 功能特性

- **元数据同步**：支持与 Jellyfin 媒体库双向同步元数据
- **用户绑定**：可以绑定指定 Jellyfin 用户，做个性化配置
- **头像自动补全**：自动识别并补全演员缺失的头像
- **状态监控**：实时监控 Jellyfin 服务器的运行状态

## 注意事项

:::warning[服务地址配置]
别用 `localhost` 或 `127.0.0.1` 作为服务地址，特别是在 Docker 等容器环境里，否则可能连不上。
:::

:::info[自动补全前提]
开启自动补全之前，请确保 AMMDS 里已经存了对应演员的头像，不然补全功能可能用不了。
:::

:::tip[API 密钥权限]
建议给 AMMDS 单独创建一个 API 密钥，只给必要的权限，这样更安全。
:::
