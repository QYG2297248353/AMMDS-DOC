## 前言

AMMDS v1.1.0 版本于 2025 年 11 月正式发布，这是继 3 月 v1.0.0 公测以来的第一个重大功能更新。本文将从实际使用角度出发，对 v1.1.0 的新特性进行深度测评，分享真实的配置建议和使用体验。

## 一、AMMDS 助手管理页面

### 功能概述

v1.1.0 最引人注目的新增功能是 **AMMDS 助手**（AMMDS Assistant）管理页面。这是 AMMDS 面向管理员提供的系统级管理面板，整合了多项运维功能。

### 核心能力

**系统健康检查**：助手页面内置了健康检查功能，可以一键检测以下各项的状态：

- 数据库连接状态（SQLite）
- 各刮削插件的 API 连通性
- 已认证站点的可达性
- 磁盘空间使用率
- 系统运行时间

**运行状态总览**：

```
系统状态
├── 服务运行时间：15 天 7 小时 32 分
├── 数据库：✅ 正常（37.2 MB）
├── TMDB 插件：✅ 已连接
├── JavBus 插件：✅ 已连接
├── M-Team 站点：✅ 已认证
├── KamePT 站点：✅ 已认证
├── Rousi 站点：⚠️ Cookie 即将过期
└── 磁盘使用率：42%（剩余 128 GB）
```

### 实际使用感受

助手页面的设计理念非常实用——它将原本散落在不同菜单项中的系统信息集中展示，让管理员一眼就能掌握系统整体状态。特别是 Rousi 站点的 Cookie 过期提醒功能，非常贴心。PT 站点的 Cookie 有效期通常只有 30 天，如果不及时更新，认证就会失效，这个提醒功能可以避免很多麻烦。

**改进建议**：希望未来版本可以增加自定义健康检查间隔和通知阈值，以及支持更多的系统指标监控（如 CPU 负载、内存使用率等）。

## 二、3D Logo 登录页面

### 视觉升级

v1.1.0 对登录页面进行了大幅度的视觉改造，引入了 **3D Logo 动画效果**。此前 AMMDS 的登录页面采用标准的平面 Logo，虽然简洁但缺乏辨识度。新的 3D Logo 在页面加载时呈现旋转动画，配合动态背景光效，整体观感向现代 SaaS 产品看齐。

### 前端技术

据观察，3D Logo 的实现采用了 Three.js 或 CSS 3D Transforms 技术（具体取决于版本），利用了 GPU 硬件加速，在主流浏览器上运行流畅，没有明显的性能开销。登录页面的加载速度相比上一版本没有明显下降。

### 用户反馈

在实际体验中，3D Logo 的视觉效果确实令人印象深刻，特别是在首次部署后打开登录页面时。它传达了一种"专业、现代"的产品印象，有助于提升用户对 AMMDS 的品牌认知。

**需要注意**：对于服务器资源紧张的环境（如低配 NAS），3D 动画可能会增加 CPU 负载。建议在系统设置中保留一个"禁用 3D 动画"的选项，以适应不同性能的硬件环境。

## 三、JavBus 站点支持的刮削效果

### 背景

JavBus 是国内影视爱好者常用的日本影视资源数据库，提供了丰富的元数据信息。在 v1.1.0 中，AMMDS 正式集成了 JavBus 作为刮削数据源。

### 刮削效果实测

我们用一个典型的资源文件来测试 JavBus 刮削效果：

**测试文件**：`FC2-PPV-1234567.mp4`

**刮削结果**：
- 影片标题：✅ 准确识别日文和中文标题
- 封面海报：✅ 高分辨率海报下载
- 演员信息：✅ 完整演员列表及头像
- 发行日期：✅ 精确到日
- 影片编号：✅ 自动识别并关联
- 剧情简介：✅ 日文简介+中文翻译
- 系列信息：✅ 系列名和系列内序号

### 多源聚合表现

AMMDS 的智能匹配算法会同时从 JavBus 和其他可用数据源获取信息，然后进行数据聚合。例如，当 JavBus 提供了演员信息而 TMDB 提供了评分数据时，AMMDS 会自动合并两者，生成最完整的元数据记录。

### 配置细节

JavBus 插件的配置非常简单：
1. 在插件管理页面启用 JavBus 插件
2. 无需额外 API Key（JavBus 是公开数据源）
3. 设置请求间隔（建议 1-2 秒，避免触发反爬机制）
4. 配置优先级（建议设为中等，让 TMDB 优先匹配欧美内容）

## 四、在线商店支持

### 功能意义

AMMDS 接入了 **在线商店**（store.lifebus.top），为支持 AMMDS 开发的用户提供了更便捷的购买渠道。

### 使用方式

访问 [足迹店铺](https://store.lifebus.top/) 即可购买相关产品和服务，支持 AMMDS 的持续发展。

### 对社区的影响

开源精神和商业可持续性并不矛盾。AMMDS 作为一个私人数据平台，持续的开发和维护需要资源支持。在线商店的接入为愿意支持项目的用户提供了一个直接的渠道，同时也让项目的商业化探索迈出了第一步。

## 五、实际使用体验总结

### 部署与升级

从 v1.0.x 升级到 v1.1.0 的过程非常平滑：

```bash
docker compose pull
docker compose up -d
```

升级后数据库自动迁移完成，所有原有配置（站点认证、插件设置、用户账户）均保留完好，没有出现兼容性问题。

### 性能表现

在 DS920+（J4125、8GB RAM）上运行一周的数据：

| 指标 | v1.0.x | v1.1.0 |
|:----|:------:|:------:|
| 内存占用 | ~180 MB | ~210 MB |
| CPU 空闲占用 | < 1% | < 2% |
| 刮削速度（单文件） | ~8 秒 | ~7 秒 |
| 页面加载时间 | ~1.2 秒 | ~1.5 秒 |

新增功能对系统资源的占用在合理范围内，整体性能表现令人满意。

### 推荐配置

经过实际测试，以下是针对不同使用场景的配置建议：

**轻量用户**（家庭影音，2-3 人）：
- Docker 参数：无需特别调整
- 启用插件：TMDB + JavBus
- 认证站点：1-2 个

**进阶用户**（多站管理，大量资源）：
- Docker 参数：增加 `AMMDS__MaxConcurrentScrapes=3`
- 启用插件：TMDB + JavBus + FANZA DMM
- 认证站点：3-5 个
- 建议开启 Telegram 通知

## 六、展望

v1.1.0 为 AMMDS 的未来发展奠定了良好的基础。从这次更新可以看出，项目团队在三个方面有清晰的规划：

1. **运维工具化**：通过助手页面降低运维复杂度
2. **视觉品牌化**：3D Logo 等设计提升产品质感
3. **生态多样化**：不断扩展刮削数据源和站点支持

期待在后续版本中看到更多创新功能，也欢迎更多用户加入到 AMMDS 社区中来，共同打造更好的私人数据平台。
