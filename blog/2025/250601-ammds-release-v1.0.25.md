# 发布 AMMDS v1.0.25 — 内置翻译与剧透模式

AMMDS v1.0.25 带来了两个实用新功能：内置翻译引擎和剧透模式。同时修复了社区反馈的多个关键问题，提升了整体稳定性。

{/* truncate */}

## 内置翻译引擎

在 v1.0.25 中，AMMDS 集成了本地化的翻译能力，用户无需依赖外部翻译服务即可完成元数据的多语言处理：

- **百度翻译接入**：通过百度翻译 API 实现中英等多种语言的互译，适合需要批量翻译影视标题、简介的场景
- **微软翻译接入**：同时支持 Azure Translator 服务，为用户提供更多选择，可根据网络环境和个人偏好灵活切换
- **翻译缓存**：已翻译的内容会缓存到本地，避免重复请求，节省 API 调用配额

这两个翻译引擎主要应用于 Metatube 刮削流程中，确保刮削到的外文元数据能够自动转换为用户习惯的语言。

## 剧透模式

新增**剧透模式**，默认开启。该功能允许用户对影视内容添加剧透标记：

- 在影视详情页面中，带有剧透标记的内容会被模糊处理
- 用户需要手动点击才能展开查看具体内容
- 这对于整理新上映影视或热门剧集的用户来说非常实用，可以有效避免无意中看到剧情关键信息

## 问题修复

- **修复 Metatube 翻译问题**：解决了 Metatube 刮削过程中翻译引擎调用失败导致元数据缺失的问题
- **修复影视订阅失败问题**：部分用户反馈的影视订阅任务因网络波动或站点响应异常而中断的问题已得到修复
- **刮削日志优化**：重构了刮削日志的输出格式，增加结构化字段，便于排查刮削失败原因
- **演员刮削逻辑优化**：优化了演员信息的刮削流程，减少重复请求，提升了刮削效率和准确性

## 其他改进

- 优化了翻译引擎的并发控制，避免因请求过快被限流
- 提升了剧透模式与现有 UI 组件的兼容性
- 改进了配置页面的交互体验
