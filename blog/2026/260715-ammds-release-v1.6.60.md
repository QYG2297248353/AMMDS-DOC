# 发布 AMMDS v1.6.60

在公测一周年之后，AMMDS 以 v1.6.60 版本开启了新的征程。本版本在功能完善、性能优化和问题修复上持续发力，进一步巩固了 AMMDS 的稳定性。

{/* truncate */}

## 新功能与改进

### AMMDS Launcher 正式版

Windows 原生客户端 AMMDS Launcher 经过三个月的公测，正式版随 v1.6.60 一同发布。相比公测版本，正式版在以下方面进行了改进：

- **自动更新模块**重写，支持增量更新，减少下载量
- 托盘菜单增加了**任务状态预览**功能
- 优化了**内存管理**，长时间运行的内存占用降低约 30%

### 飞牛 FnOS FPK 安装包

针对飞牛 FnOS 用户，新增了 FPK 格式的离线安装包，可通过 FnOS 应用商店直接导入安装，无需配置 Docker 环境。

### 插件体系增强

- **插件 SDK v1.0 正式发布**：API 稳定，文档完善，欢迎社区开发者贡献插件
- **新增站点检索插件接口**：在原有三类插件基础上，新增第四类——站点检索插件
- **插件热加载优化**：启用/禁用插件不再需要重启 AMMDS 服务

### 榜单功能增强

- 榜单数据刷新策略优化，减少对数据源的请求频率
- 新增**自定义榜单**功能，用户可以根据标签、演员、资源类型等维度创建个性化榜单

## Bug 修复

- **刮削任务死锁**：修复了高并发刮削场景下任务队列死锁的问题
- **NFO 编码问题**：修复了部分中文元数据写入 NFO 文件时编码错误导致媒体服务器无法正确读取的问题
- **演员头像丢失**：修复了更新演员信息后已有头像被错误覆盖的问题
- **站点认证过期检测**：修复了部分情况下 Cookie 过期但系统未及时告警的问题
- **日志轮转**：修复了长时间运行时日志文件无限增长导致磁盘空间耗尽的问题

## 升级指南

### Docker 部署

```bash
# 拉取最新镜像
docker pull ghcr.io/mengshou/ammds:latest  # 国内用户可替换为 qyg2297248353/ammds:latest（Docker Hub）

# 重新启动容器
docker compose down
docker compose up -d
```

### Windows Launcher 用户

AMMDS Launcher 会自动检测并推送更新。你也可以在托盘菜单中点击"检查更新"手动触发。

### 注意事项

1. v1.6.60 的数据库结构有微小变更，升级过程会自动迁移，请确保备份数据后再升级
2. 本次升级重新组织了插件目录结构，启用自定义插件的用户请在升级后检查插件配置
3. 自动迁移完成后，建议在管理面板的"系统信息"页面确认版本号已更新为 `v1.6.60`

## 已知问题

- 在极低配置的 Windows 设备上，AMMDS Launcher 首次启动时可能会稍慢（约 10-15 秒），后续启动速度正常
- FnOS FPK 包在部分网络环境下下载可能失败，建议使用代理或手动下载安装
