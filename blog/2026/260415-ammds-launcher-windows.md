## 从 Web 到桌面

自 AMMDS 公测以来，我们一直以 Web 管理面板 + Docker 后端的形式提供服务。这种方式虽然灵活可靠，但对于大量使用 Windows 作为主力系统的用户来说，存在天然的体验鸿沟——需要额外学习 Docker 的使用方法，配置环境变量，管理容器生命周期。

现在，这一切都已成为过去。

**AMMDS Launcher**——Windows 原生桌面客户端——正式发布了！它采用 EXE 安装包形式，为用户提供了一键安装、托盘运行、自动启动等桌面级体验。

## 技术架构

AMMDS Launcher 并不是简单地将 Web 界面打包进 Electron 容器，而是对整体架构进行了重新设计：

**后端**：核心引擎仍使用 Go 语言编写，保持与 Docker 版本一致的高性能和低资源占用。后端以 Windows 服务的形式运行在系统中。

**前端**：桌面 UI 使用 React + Tauri 构建，相比 Electron 方案，Tauri 在内存占用和启动速度上具有显著优势。同时保留了 Web 管理面板的全部功能。

**通信层**：前后端之间通过本地 HTTP API 进行通信，默认绑定在 `127.0.0.1` 以确保安全。为避免端口冲突，Launcher 会自动检测并分配可用端口。

## 安装体验优化

### 管理员权限

AMMDS Launcher 安装包在安装过程中会自动请求管理员权限。这主要是因为 AMMDS 需要访问本地文件系统中的媒体资源目录，以及在某些配置下需要监听特定端口。我们最小化了权限需求——仅在安装和配置阶段需要管理员权限，日常运行以普通用户权限即可。

### 自动启动

安装完成后，AMMDS Launcher 会提供开机自启动选项。开启后，Windows 启动时 AMMDS 会在后台静默启动，用户无需手动操作即可随时通过托盘图标访问管理面板。自动启动功能支持延迟启动（延迟 30 秒），避免影响系统开机速度。

### 托盘管理

AMMDS Launcher 最小化后不会消失，而是收缩到 Windows 系统托盘中。右键点击托盘图标，可以快速执行以下操作：

- **打开管理面板**：在默认浏览器中打开 AMMDS 管理界面
- **查看运行状态**：快速查看刮削任务、系统资源占用等关键指标
- **暂停/恢复**：在不退出程序的情况下暂停所有刮削任务
- **查看日志**：快速访问运行日志，便于排查问题
- **退出**：完全退出 AMMDS Launcher

## 相比 Docker 版本的区别

AMMDS Launcher 与 Docker 版本共享同一套核心引擎，但在以下几个方面有所差异：

| 特性 | Docker 版本 | Windows Launcher |
|:----|:-----------|:----------------|
| 部署方式 | Docker CLI / Compose | EXE 安装程序，一键安装 |
| 平台支持 | 所有支持 Docker 的系统 | Windows 10/11 64位 |
| 自动更新 | 手动 pull 镜像 | 内置自动更新模块 |
| 资源占用 | 取决于容器配置 | 优化了 Windows 下的内存管理 |
| 数据目录 | 通过 volume 挂载 | 默认 `%LOCALAPPDATA%/AMMDS` |
| 日志管理 | Docker logs | Windows 事件日志 + 文件日志 |

## 下载方式

AMMDS Launcher 目前提供以下下载渠道：

1. **GitHub Releases**：访问 [AMMDS GitHub 仓库](https://github.com/mengshou/ammds) 的 Releases 页面，下载最新版本的 `.exe` 安装包
2. **飞牛 FnOS 应用商店**（即将上线）：FnOS 用户可直接在应用商店中搜索 AMMDS 进行安装

安装包大小约 80MB，支持 Windows 10 1903 及以上版本和 Windows 11。

## 升级与卸载

**升级**：AMMDS Launcher 内置了自动更新功能。当新版本可用时，托盘图标会显示更新提示，用户点击确认后即可完成自动升级，无需手动下载安装包。

**卸载**：通过 Windows"添加或删除程序"即可完整卸载 AMMDS Launcher，系统会自动清理相关文件和配置。

## 结语

AMMDS Launcher 的发布，是 AMMDS 在降低使用门槛道路上迈出的重要一步。我们希望更多的 Windows 用户能够零门槛地体验到 AMMDS 的强大功能。未来，我们还会考虑推出 macOS 版本和移动端 App，让 AMMDS 覆盖更多的使用场景。
