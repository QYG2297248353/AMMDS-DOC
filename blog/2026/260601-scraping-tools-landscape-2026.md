## 生态变化概览

过去一年半，刮削工具生态经历了几个重要变化：

- **MDC-NG 正式发布**：Rust 重构的刮削引擎，代表刮削工具进入现代化 Web 架构时代
- **SakuraMedia 崛起**：Flutter 跨平台 NSFW 平台，拓展了"观影"而非"管理"的新品类
- **MetaTube 社区持续活跃**：插件式刮削方案不断完善，Server + Plugin 模式得到验证
- **AMMDS v1.6 系列成熟**：从刮削工具演进为私人数据平台，插件生态初步成型
- **MDCz 稳定迭代**：延续 MDCx 的 PyQt 路线，代码质量显著提升
- **Stash 社区保持活力**：作为 NSFW 领域的经典选择，社区插件数量稳步增长

## 新工具深度观察

### MDC-NG：Rust 时代的刮削标杆

MDC-NG 是 2025-2026 年度最值得关注的新工具之一。它的出现验证了几个重要趋势：

- **Rust 在刮削场景的可行性**：高性能 + 低资源占用 + 内存安全，非常适合在 NAS 等资源受限设备上运行
- **Web 界面取代桌面界面**：Next.js 构建的现代 UI，操作体验已接近桌面应用
- **服务化趋势**：从"用的时候打开"到"始终运行"的转变

MDC-NG 镜像 < 300MB，运行内存 < 300MB，更新仅需约 60MB——这些数字设定了一个新的技术基准，也是对 AMMDS 等技术栈不同的工具的激励。

### SakuraMedia：重新定义"观影"

SakuraMedia 的价值不在于刮削，而在于**观影体验**。它解决了传统刮削工具长期忽略的问题：

- 内容最终是给人看的，不是给管理面板看的
- Flutter 原生客户端的观影流畅度远超 Web 播放器
- 订阅 + 自动下载的模式更接近 Netflix 等流媒体平台的使用习惯

SakuraMedia 的出现拓宽了我们对"刮削工具"的认知——刮削只是手段，观影才是目的。

### MetaTube 进入成熟期

MetaTube 在 2025-2026 年间进行了多项重要更新：

- 数据库缓存机制优化，首次查询速度提升 40%
- 新增多个数据源适配器，覆盖更广泛的影片类型
- 插件 API 趋于稳定，社区贡献活跃

MetaTube 的最大价值在于证明了"刮削服务化"的可行性——将刮削能力从媒体服务器中分离出来，作为一个独立的服务运行。

## 工具选择决策树

```
你的需求是什么？
├── 只需要刮削 → 你有媒体服务器吗？
│   ├── 有 Jellyfin/Emby → MetaTube（轻量）或 AMS 插件（深度集成）
│   └── 没有 → MDC-NG（现代）或 MDCz（稳定桌面）
├── 需要管理 + 观影 → 内容类型？
│   ├── 仅 NSFW → SakuraMedia（体验优先）或 Stash（功能优先）
│   └── 通用 + NSFW 混合 → AMMDS
└── 需要全链路自动化 → AMMDS（唯一从检索到推送的完整方案）
```

## 展望 2026-2027

未来一年，刮削工具生态可能会出现以下趋势：

1. **AI 能力深度整合**：AI 人脸识别现在已经是标配，下一步可能是 AI 自动生成元数据、AI 场景分类
2. **跨工具互操作性增强**：支持导入/导出标准格式的元数据，工具之间不再是孤岛
3. **服务化成为主流**：越来越多的刮削工具从桌面应用转向服务端应用
4. **生态整合加速**：用户倾向于使用更少但更全面的工具，单一功能的工具需要找到差异化定位
5. **移动端体验升级**：随着 SakuraMedia 等 Flutter 应用的带动，移动端观影和管理需求将被更多工具重视

## 结语

2026 年的刮削工具生态比以往任何时候都更加丰富和多元。新工具（MDC-NG、SakuraMedia）带来了新的技术路线和产品理念，老工具（Stash、MetaTube、MDCz）在各自领域持续深耕，平台型工具（AMMDS）则试图整合全链路体验。

无论你是刚入门的用户，还是经验丰富的媒体库管理者，现在的选择都比以往任何时候更多——找到最适合自己的工具，享受媒体管理的乐趣。
