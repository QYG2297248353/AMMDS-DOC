## 前言

截至 2026 年 7 月，AMMDS 已经迭代至 v1.6.60 版本。经过一年半的持续演进，AMMDS 已经从一个单一功能的工具，发展成为一个拥有完整生态的私人数据平台。本文将系统梳理 AMMDS 的生态全景，帮助用户全面了解当前版本的功能体系。

## 一、插件体系

插件是 AMMDS 生态的核心，它让平台具备了高度的可扩展性。目前 AMMDS 的插件体系分为四大类：

### 1.1 刮削插件（Scraper Plugins）

刮削插件负责从各类数据源获取影视资源的元数据。这是 AMMDS 最核心的插件类型。

**Metatube 刮削插件**
Metatube 是一个综合性的元数据源，覆盖了广泛的电影、剧集、动漫资源。它的特点是数据更新及时，对冷门资源也有不错的覆盖率。在 AMMDS 中，Metatube 通常被配置为欧美影视的主要刮削源之一。

**ThePornDB 刮削插件**
面向成人内容的数据源插件，提供完善的元数据刮削能力。该插件采用了专门的匹配算法，能够准确识别资源内容并获取相关元数据。

**FANZA DMM 刮削插件**
FANZA DMM 是日本最大的数字内容平台之一，面向日本影视资源提供高质量的元数据。该插件的加入极大地提升了 AMMDS 在日本影视领域的刮削质量，包括完整的演员信息、高分辨率封面、详细的影片描述等。

**TMDB 刮削插件（内置）**
The Movie Database（TMDB）是最广泛使用的开放影视数据库，作为 AMMDS 的默认刮削源，覆盖欧美影视的绝大部分需求。

**JavBus 刮削插件**
自 v1.1.0 引入以来，JavBus 插件已经成为日本影视资源用户的首选。它无需 API Key，配置简单，刮削质量稳定。

### 1.2 媒体服务器插件（Media Server Plugins）

媒体服务器插件负责 AMMDS 与各类媒体服务器之间的通信和数据同步。

**Emby 插件**
通过 Emby API 实现与 Emby 媒体服务器的对接。AMMDS 刮削完成的元数据可以直接同步到 Emby 媒体库中，无需手动刷新。

**Jellyfin 插件**
与 Emby 插件类似，面向开源免费的 Jellyfin 媒体服务器。支持元数据同步、媒体库刷新和状态更新。

### 1.3 站点检索插件（Site Search Plugins）

站点检索插件是 AMMDS 与 PT 站点之间的桥梁，支持在多个站点中并行检索资源。

**NexusPHP 站点检索插件**
基于 NexusPHP 框架的通用检索插件，对所有基于 NexusPHP 的 PT 站点提供统一的检索接口。

**M-Team 站点检索插件**
针对 M-Team（馒头）站点的专属检索插件，优化了检索效率和结果展示。

**Rousi 站点检索插件**
针对 Rousi（肉丝）站点的专属检索插件，支持高级筛选和排序。

### 1.4 推送通知插件（Notification Plugins）

推送通知插件负责将 AMMDS 的运行状态、刮削结果等信息推送给用户。

**Telegram 推送插件**
通过 Telegram Bot 发送通知。支持自定义通知模板，可以配置需要推送的事件类型（如刮削完成、站点认证过期、系统错误等）。

**WebHook 推送插件**
提供通用的 WebHook 接口，用户可以将其与自己的消息系统（如 Slack、Discord、钉钉、企业微信等）对接。

## 二、榜单体系

AMMDS 的榜单体系是帮助用户发现和管理热门资源的重要功能。榜单模块从多个维度聚合数据，为用户提供决策参考。

### 榜单类型

**热门榜单**
聚合各 PT 站点和刮削数据源的热门资源排行。用户可以通过热门榜单快速发现当前最受关注的影视内容。

**新资源榜单**
实时展示最新加入的资源，帮助用户第一时间获取新内容。

**精选推荐**
基于用户的历史刮削数据和收藏行为，推荐可能感兴趣的资源。

### 榜单的个性化

AMMDS 的榜单系统支持用户根据自身偏好进行定制：
- 按资源类型筛选（电影、剧集、动漫、纪录片等）
- 按数据源筛选（仅显示特定站点的数据）
- 按时间范围筛选（今日、本周、本月）

## 三、站点认证体系

站点认证是 AMMDS 区别于普通刮削工具的核心差异化功能。

### 当前支持的站点

截至 v1.6.60，AMMDS 已支持以下站点的认证对接：

| 站点 | 类型 | 认证方式 | 功能 |
|:----|:----|:--------|:----|
| M-Team（馒头） | 综合 PT | Cookie/Passkey | 资源检索、下载管理 |
| KamePT | 影视 PT | Cookie | 资源检索 |
| Rousi（肉丝） | 综合 PT | Cookie | 资源检索、状态监控 |
| 更多站点持续接入中... | - | - | - |

### 认证管理功能

**统一认证中心**
可以在一个界面中管理所有已认证站点的状态、查看 Cookie 有效期、进行重新认证操作。

**健康监控**
系统会定期检测各站点的连通性和认证状态，并在发现问题时通过推送插件通知管理员。

**多站点并行检索**
当用户发起资源检索时，AMMDS 会在所有已认证且状态正常的站点中并行执行搜索，然后汇总返回结果。

## 五、使用场景推荐

根据不同的用户群体，AMMDS 提供了多种使用方案：

### 场景一：个人影音爱好者

**需求**：整理本地影视资源，获得精美的海报墙
**推荐配置**：
- 部署方式：Docker 或 Windows 安装包
- 刮削插件：TMDB + JavBus
- 媒体服务器：Jellyfin（免费）
- 通知方式：无需配置

### 场景二：PT 站点多站用户

**需求**：管理 2-5 个 PT 站点的资源检索和下载
**推荐配置**：
- 部署方式：Docker on NAS
- 刮削插件：TMDB + JavBus + FANZA DMM
- 站点认证：所有已注册的 PT 站点
- 媒体服务器：Emby 或 Jellyfin
- 通知方式：Telegram

### 场景三：小型团队/家庭共享

**需求**：多人共享影音库，统一管理
**推荐配置**：
- 部署方式：Docker on 高性能 NAS 或 VPS
- 刮削插件：全部启用
- 站点认证：全部可用站点
- 媒体服务器：Emby（多用户管理更完善）
- 通知方式：WebHook → 企业微信/钉钉

## 六、未来展望

站在 v1.6.60 的时间节点，AMMDS 的未来仍然充满想象空间。以下是我们规划中的发展方向：

### 短期规划（v1.7.x - v1.8.x）

1. **插件 SDK 开放**：发布官方的插件开发 SDK 和文档，鼓励社区开发者贡献自定义插件
2. **AI 智能识别**：引入 AI 模型，对无法自动识别的资源进行智能分类和匹配
3. **移动端应用**：开发 AMMDS 移动端 App，方便随时管理刮削任务和查看系统状态
4. **更多站点接入**：持续扩展 PT 站点和其他数据源的对接

### 中期规划（v2.0）

1. **分布式架构**：支持多节点部署，提升大规模资源管理场景下的性能
2. **数据可视化**：提供更丰富的数据统计和可视化报表
3. **自动化工作流**：支持自定义刮削工作流，满足复杂的业务需求
4. **国际化**：完善多语言支持，拓展海外用户群体

### 长期愿景

让 AMMDS 成为私人数据管理的首选平台，无论是影视资源、音乐收藏、电子书还是个人数据，都能在 AMMDS 上得到高效、智能的管理体验。

## 结语

AMMDS 的生态体系正在快速成长。从最初的简单工具到如今拥有插件、榜单、站点认证的完整平台，每一步都离不开社区用户的反馈和支持。

如果你还没有尝试过 AMMDS，现在是最好的时机——通过 Docker 一键部署，几分钟内就能体验到完整的私人数据管理功能。如果你已经是 AMMDS 的用户，欢迎关注后续版本的更新，我们一起让 AMMDS 变得更好。
