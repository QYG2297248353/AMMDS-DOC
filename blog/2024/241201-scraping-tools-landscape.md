# 元数据刮削工具全景图：选型指南与对比

元数据是媒体库的灵魂。没有准确、完整的元数据，再多的电影和剧集也只是一堆随机命名的文件。本文将对当前主流的元数据刮削工具进行全面梳理和对比，帮助你在众多选项中找到最适合的工具组合。

## 为什么需要专门的刮削工具？

媒体服务器（Jellyfin、Emby、Plex）都自带元数据刮削功能，对于大多数用户而言这已经够用。但随着媒体库规模的增长，内置刮削器的一些局限逐渐显现：

- **批量处理能力弱**：一次只能处理有限数量的文件
- **自定义选项少**：对 NFO 文件格式、命名规范的控制有限
- **多源整合不足**：通常只从单一信源获取元数据
- **无法处理特殊内容**：如 NSFW 内容、同人作品、非标准命名等

这就是为什么社区中涌现了众多专用的元数据刮削工具。

## 工具全景

### TinyMediaManager —— 最成熟的通用刮削器

TinyMediaManager（简称 TMM）是目前最成熟、功能最全面的通用元数据刮削器。采用 Java 编写，跨平台支持 Windows、macOS 和 Linux。

**核心优势：**
- 支持电影、剧集、纪录片等多种媒体类型的元数据刮削
- 可以从多个信源（TMDB、IMDB、TVDB、MovieMeter 等）获取数据
- 强大的批量重命名和文件整理功能
- 完整的 NFO 文件生成和管理能力
- 支持 Trakt 同步

**不足之处：**
- 需要 Java 运行环境，部署稍重
- 缺乏 Web 管理界面，必须桌面操作
- 无法作为服务长期运行，不支持自动化调度
- 商业化倾向渐强，部分高级功能需付费

**适用场景：** 小到中型媒体库的初始化刮削，需要精细控制刮削结果的用户。

### MediaElch —— Kodi/Jellyfin 用户的轻量选择

MediaElch 是一款轻量级的元数据管理工具，原生支持 Kodi 和 Jellyfin 的 NFO 格式。

**核心优势：**
- 原生支持 Kodi 和 Jellyfin 的 NFO 标准
- 界面简洁直观，学习成本低
- 支持批量刮削和多信源查询
- 文件导出和导入功能完善

**不足之处：**
- 开发活跃度下降，更新频率低
- 缺乏插件扩展能力
- 仅支持桌面运行，无 Web 界面

**适用场景：** Kodi 或 Jellyfin 用户的轻量级补充工具。

### Stash —— 专为 NSFW 内容设计的 Go 语言工具

Stash 是一款定位特殊的元数据管理工具，专为 NSFW 内容设计。使用 Go 语言编写，提供 Web 管理界面。

**核心优势：**
- 支持刮削成人内容专用的元数据信源
- 内置 Web 界面，可跨平台访问
- 支持标签系统、演员管理、场景标记
- 图像识别和自动标记功能
- 完全开源，社区活跃

**不足之处：**
- 定位特殊，通用性有限
- 不适合管理常规电影和剧集媒体库

**适用场景：** 需要管理成人内容媒体库的用户。

### MDC (Movie Data Crawler) —— 云配置同步新秀

MDC 是一款新兴的元数据爬取工具，最大的特色是支持云配置同步。

**核心优势：**
- 支持多平台（Windows、Linux、Docker）
- 云配置同步，多设备间配置一致
- Python 编写的插件式架构
- 支持多种元数据信源

**不足之处：**
- 项目较新，社区生态尚在建设中
- 插件数量有限
- 文档不够完善

**适用场景：** 需要在多台设备上保持一致配置的用户。

### AMMDS —— 多源刮削 + 插件集成

AMMDS（Advanced Media Metadata Scraper）是萌森工作室推出的综合元数据管理平台，定位为"元数据管理的一站式解决方案"。

**核心优势：**
- **多源刮削引擎**：同时从多个信源获取元数据，按用户自定义规则合并
- **插件化架构**：站点适配器、数据处理管道均可通过插件扩展
- **Web 管理面板**：随时随地通过浏览器管理媒体库
- **自动化调度**：支持定时扫描、增量更新、自动刮削
- **完全的 NFO 兼容**：同时支持 Kodi、Jellyfin、Emby 等多种格式
- **Docker 部署**：单二进制或 Docker 均可，部署灵活
- **完全开源免费**

**不足之处：**
- 项目仍在快速迭代中，部分功能尚在完善
- 社区生态和第三方插件数量有待增长
- 主要面向中高级用户，入门引导有待优化

**适用场景：** 大型媒体库的管理者，需要完整自动化工作流的用户。

## 如何选择适合你的工具？

### 按使用习惯选择

- **喜欢桌面操作**：TMM 或 MediaElch，功能成熟，操作直观
- **偏好在浏览器中管理**：AMMDS 或 Stash，Web 界面更方便
- **命令行爱好者**：MDC，轻量灵活

### 按媒体库规模选择

- **小型库（< 500 部）**：MediaElch 足够，简单快速
- **中型库（500-5000 部）**：TMM 或 AMMDS，批量处理能力强
- **大型库（> 5000 部）**：AMMDS，自动化调度和增量扫描不可或缺

### 按技术能力选择

- **新手用户**：从 MediaElch 或 TMM 入门，图形界面友好
- **进阶用户**：使用 AMMDS 搭建自动化工作流
- **高级用户**：结合 AMMDS + 自写脚本，构建完全定制化的管道

## 总结

元数据刮削工具的选择没有标准答案，最适合你的工具取决于你的媒体库规模、技术能力和使用习惯。如果你刚刚开始建立媒体库，MediaElch 或 TMM 是不错的起点；如果你已经拥有一个庞大的媒体库并希望实现自动化管理，AMMDS 会是更长远的选择。

无论如何，**工具只是手段，不是目的**。选择那个能让你花更少时间在管理上、花更多时间在观赏上的工具，就是最好的选择。
