# 飞牛 FnOS 배포 가이드

飞牛 FnOS는 고급 사용자 팀이 개발한 NAS 시스템으로, AI 로컬 모델을 통합하여 가정 및 기업에 효율적이고 안전한 스토리지 솔루션을 제공합니다. 본 가이드는 AMMDS를飞牛 FnOS 환경에 배포하는 과정을 자세히 설명합니다.

<!-- truncate -->

## 배포 방식 개요

AMMDS는飞牛 FnOS에서 두 가지 배포 방식을 제공합니다:

- **FPK 오프라인 애플리케이션 설치**：飞牛 FnOS 애플리케이션 센터를 통해 수동 설치，대부분의 사용자에게 적합
- **Docker 설치**：Docker 컨테이너를 통해 배포，컨테이너 기술에 익숙한 사용자에게 적합

## FPK 오프라인 애플리케이션 설치

### 다운로드

![AMMDS FPK 다운로드 페이지](/img/deploy/install-fnos-fpk-00.png)

#### 공식 배포 주소

[AMMDS Offline FnNas GitHub Releases](https://github.com/QYG2297248353/AMMDS-Offline-FnNas/releases)

#### 사용 가능한 설치 패키지

| 파일 이름                  | 타입 | 설명                 |
| ----------------------- | ---- | -------------------- |
| `AMMDS-Offline-FnNas.fpk` | FPK  | AMMDS 오프라인 설치 패키지，飞牛 FnOS 시스템에适用 |

#### 다운로드 방식

![AMMDS FPK 다운로드 옵션](/img/deploy/install-fnos-fpk-01.png)

로컬 컴퓨터에 설치 패키지를 다운로드하거나，직접飞牛 FnOS 장치에서 다운로드할 수 있습니다.

### 설치 단계

![AMMDS FPK 설치 인터페이스](/img/deploy/install-fnos-fpk-02.png)

1. **설치 프로세스 시작**

   애플리케이션 센터를 열고，왼쪽 아래의**수동 설치**를 클릭한 다음，다운로드한 FPK 파일을 선택하고**설치**를 클릭합니다.

![AMMDS FPK 설치 확인](/img/deploy/install-fnos-fpk-03.png)

2. **설치 확인**

   설치 확인 화면에서**동의**를 클릭하여 설치 프로세스로 들어갑니다.

![AMMDS FPK 라이선스 설명](/img/deploy/install-fnos-fpk-04.png)

3. **라이선스 설명 읽기**

   설치 라이선스 설명을 자세히 읽고，관련 조건을 이해합니다.

![AMMDS FPK 설치 위치 선택](/img/deploy/install-fnos-fpk-05.png)

4. **설치 위치 선택**

   애플리케이션 설치 디스크 위치를 선택합니다. 공간이 충분한 디스크를 선택하는 것이 좋습니다.

![AMMDS FPK 관리자 비밀번호 설정](/img/deploy/install-fnos-fpk-06.png)

5. **관리자 비밀번호 설정**

   AMMDS 관리 인터페이스의 로그인 비밀번호를 설정합니다. 강력한 비밀번호를 사용하는 것이 좋습니다.

![AMMDS FPK 액세스 포트 설정](/img/deploy/install-fnos-fpk-07.png)

6. **액세스 포트 설정**

   AMMDS의 액세스 포트를 설정합니다. 기본 포트는 9523입니다.
   서비스 약관을 읽고，**동의**를 클릭합니다.

![AMMDS FPK 설치 시작](/img/deploy/install-fnos-fpk-08.png)

7. **설치 시작**

   마지막 설치 확인 화면에서**확인**을 클릭하여 설치를 시작합니다.

![AMMDS FPK 설치 진행 상황](/img/deploy/install-fnos-fpk-09.png)

8. **설치 완료 대기**

   설치 과정 중 진행률 표시줄이 표시됩니다. 설치가 완료될 때까지 기다려 주십시오. 오류 메시지가 나타나면，메시지에 따라 처리하거나，이전 버전의 FPK 설치 패키지를 사용해 보십시오.

![AMMDS FPK 설치 완료](/img/deploy/install-fnos-fpk-10.png)

9. **설치 성공 확인**

   설치가 완료되면，설치된 애플리케이션 목록에서 AMMDS 애플리를 찾을 수 있습니다.

### 설치 디렉터리

![AMMDS 설치 디렉터리](/img/deploy/install-fnos-fpk-11.png)

파일 관리에서，애플리케이션 파일에서 AMMDS 애플리케이션의 설치 디렉터리를 찾을 수 있습니다.

## Docker 설치

### 이미지 다운로드

![AMMDS Docker 이미지 검색](/img/deploy/install-fnos-docker-00.png)

1. **이미지 검색**

   Docker 애플리를 열고，초기화를 완료한 후，왼쪽 메뉴의**이미지 저장소**를 클릭하고 AMMDS 이미지를 검색합니다.

2. **이미지 선택**

   다운로드 수가 가장 많은 이미지를 선택합니다. 전체 이미지 이름은 `qyg2297248353/ammds`입니다.

![AMMDS Docker 이미지 버전 선택](/img/deploy/install-fnos-docker-01.png)

3. **버전 선택**

   기본 버전은 `latest`입니다. 장치가 구형 장치인 경우 `latest` 버전을 지원하지 않을 수 있으며，태그 끝이 `ol8`인 버전을 선택해야 합니다.

![AMMDS Docker 이미지 다운로드 완료](/img/deploy/install-fnos-docker-02.png)

4. **다운로드 완료 대기**

   로컬 이미지에서 다운로드 진행 상황을 확인할 수 있으며，다운로드가 완료되면 실행 버튼이 나타납니다.

### 컨테이너 생성 및 구성

![AMMDS Docker 컨테이너 생성](/img/deploy/install-fnos-docker-03.png)

1. **컨테이너 생성**

   **실행** 버튼을 클릭하여 배포를 시작합니다.
   컨테이너 이름은 사용자 정의，예를 들어 `ammds`입니다.
   장치 성능과 요구 사항에 따라 적절한 리소스를 할당합니다.
   **부팅 시 자동 실행** 기능을 켤 수 있습니다.

![AMMDS Docker 포트 설정](/img/deploy/install-fnos-docker-04.png)

2. **포트 설정**

   고급 설정에서 포트 매핑을 구성합니다:
   - 기본 컨테이너 포트는 80(TCP)입니다.
   - 액세스 포트를 9523(또는 기타 사용 가능한 포트)으로 변경합니다.

![AMMDS Docker 스토리지 설정](/img/deploy/install-fnos-docker-05.png)

3. **스토리지 설정**

   지속성 스토리지를 위해 로컬 디렉터리를 컨테이너에 마운트하는 것이 좋습니다:
   - 기본적으로飞牛는 통합 임시 스토리지 디렉터리를 생성하지만，이 디렉터리는 시스템에 의해 쉽게 정리될 수 있습니다.
   - 해당 디렉터리를 수동으로 생성하고 컨테이너에 마운트하는 것이 좋습니다.

![AMMDS Docker 정보 확인](/img/deploy/install-fnos-docker-06.png)

4. **구성 확인**

   모든 구성 정보를 확인한 후，**생성 후 시작** 옵션을 선택합니다.

### 실행 및 관리

![AMMDS Docker 컨테이너 실행 상태](/img/deploy/install-fnos-docker-07.png)

1. **실행 상태 확인**

   컨테이너 목록에서 AMMDS 애플리케이션이 실행 중인 것을 확인할 수 있습니다. 오른쪽의 세 개 점을 통해 빠른 작업을 수행할 수 있습니다.

![AMMDS Docker 애플리케이션 세부 정보](/img/deploy/install-fnos-docker-08.png)

2. **애플리케이션 세부 정보 보기**

   컨테이너를 클릭하여 세부 정보 페이지로 들어가，애플리케이션의 자세한 정보를 확인합니다.

![AMMDS Docker 실행 로그](/img/deploy/install-fnos-docker-09.png)

3. **실행 로그 보기**

   **로그 보기**를 클릭하여 애플리케이션 실행 상태와 다운로드 진행 상황을 확인할 수 있습니다.

   :::info 최초 시작 안내
   최초 설치 시，시스템은 필요한 리소스 파일을 자동으로 다운로드하므로，일정 시간이 필요합니다. 로그에서 다운로드 진행 상황이 표시됩니다.
   :::

## 설치 완료

![AMMDS 설치 완료 인터페이스](/img/deploy/install-fnos-finish.png)

설치가 완료되면，브라우저를 통해 AMMDS 관리 인터페이스에 접속하여 모든 기능을 사용할 수 있습니다.

Docker의 오케스트레이션 모드에 익숙하다면，Compose 파일을 통해 AMMDS를 배포하여 보다 유연한 구성과 관리 경험을 얻을 수 있습니다.

## 경고 및 주의 사항

### 보안 경고

:::danger 중요 보안 안내

- **공식 출처**：공식 GitHub Releases 페이지에서만 AMMDS 설치 패키지를 다운로드하십시오. 타사 다운로드 원본을 사용하지 마십시오.
- **비밀번호 보안**：강력한 비밀번호를 설정하고 정기적으로 변경하십시오. 약한 비밀번호를 사용하지 마십시오.
- **포트 보안**：기본 포트를 변경하여 공격 위험을 줄이십시오.

:::

### 시스템 요구 사항

:::info 시스템 구성 요구 사항

#### 최소 구성

- 운영 체제：飞牛 FnOS 최신 버전
- 프로세서：Intel Core i3 또는 동등 성능
- 메모리：4GB RAM
- 디스크 공간：최소 10GB 사용 가능 공간
- 네트워크：광대역 인터넷 연결

#### 권장 구성

- 운영 체제：飞牛 FnOS 최신 버전
- 프로세서：Intel Core i5 또는 동등 성능
- 메모리：8GB RAM 이상
- 디스크 공간：20GB 사용 가능 공간
- 네트워크：안정적인 광대역 인터넷 연결

:::

### 사용 주의 사항

- **데이터 백업**：AMMDS의 구성 파일과 중요한 데이터를 정기적으로 백업하십시오.
- **리소스 관리**：디스크 공간 사용情况을 모니터링하고，필요하지 않은 리소스 파일을 적시에清理하십시오.
- **네트워크 연결**：리소스를 다운로드하고 업데이트할 때 특히 안정적인 네트워크 연결을确保하십시오.
- **시스템 유지 관리**：飞牛 FnOS 시스템을 정기적으로 업데이트하여 시스템 안정성을确保하십시오.

### 문제 해결

#### 일반적인 문제

- **설치 실패**：디스크 공간이 충분한지，네트워크 연결이 정상적인지 확인하십시오.
- **시작 이상**：실행 로그를 확인하여 자세한 오류 정보를获取하십시오.
- **접근 실패**：포트 구성이 올바른지，방화벽이 접근을 허용하는지 확인하십시오.
- **업그레이드 실패**：이전 버전을 먼저 제거한 후 새 버전을 설치해 보십시오.

#### 연락처

해결할 수 없는 문제가 발생하면，공식 GitHub 저장소에 Issue를 제출하거나 도움을 요청하십시오.

## 모범 사례

- **정기 업데이트**：최신 기능과 보안 패치를 받기 위해 AMMDS를 최신 버전으로 유지하십시오.
- **합리적인 구성**：장치 성능에 따라 AMMDS 매개변수를 합리적으로 구성하여 리소스 과도 사용을 방지하십시오.
- **안전한 사용**：공식 문서 지침을 따라 핵심 구성 파일을修改하지 마십시오.
- **커뮤니티 참여**：AMMDS 커뮤니티에 참여하여 사용 경험을 공유하고 문제에 대한 피드백을 제공하십시오.
