# 빠른 시작

:::info
AMMDS에 오신 것을 환영합니다! 이 가이드를 따라 Windows, 飞牛 Nas, Docker 등 지원되는 방법으로 AMMDS를 배포할 수 있습니다.
:::

## 🚀 배포 방법

### Windows 배포

:::note
**패키지 유형**: EXE 설치 패키지
:::

Windows 사용자는 간편한 설치 패키지를 제공합니다:

1. [AMMDS Launcher 릴리스 페이지](https://github.com/QYG2297248353/AMMDS-Launcher/releases)에서 최신 `ammds-setup.exe` 다운로드
2. 다운로드한 `ammds-setup.exe` 파일 찾기
3. 마우스 오른쪽 클릭 → **관리자 권한으로 실행**
4. 설치 마법사를 따라 진행
5. 설치 완료 후 서비스가 자동으로 시작됨

---

### 飞牛 Nas 배포

:::note
**패키지 유형**: FPK (飞牛 패키지)
:::

飞牛 Nas 사용자는 FPK 패키지를 제공하며, 飞牛 앱 스토어에 직접 가져올 수 있습니다:

1. [飞牛 Nas 릴리스 페이지](https://github.com/QYG2297248353/AMMDS-Offline-FnNas/releases)에서 최신 `AMMDS.offline.fpk` 다운로드
2. 飞牛 Nas 관리 화면에 로그인
3. **飞牛 앱 스토어** 열기
4. **로컬 패키지 가져오기** 선택 후 다운로드한 FPK 파일 선택
5. 안내에 따라 설치 완료
6. 이후 飞牛 Nas 앱 스토어에서 서비스 관리 가능

---

### Docker 배포

:::note
**패키지 유형**: Docker 이미지
:::

Docker가 설치되어 있다면 한 줄 명령어로 AMMDS를 배포할 수 있습니다:

```bash showLineNumbers title="Docker Command"
docker run -itd \
   --name AMMDS \
   -p 8080:80 \
   -v $(pwd)/data:/ammds/data \
   -v $(pwd)/db:/ammds/db \
   --restart always \
   qyg2297248353/ammds:latest
```

#### 명령어 매개변수 설명

| 옵션 | 설명 |
|--------|-------------|
| `-itd` | 컨테이너를 백그라운드에서 실행, 터미널 창을 차지하지 않음 |
| `--name AMMDS` | 컨테이너 이름을 "AMMDS"로 지정하여 관리 편리 |
| `-p 8080:80` | 로컬의 8080 포트를 컨테이너의 80 포트(웹 인터페이스)에 매핑 |
| `-v $(pwd)/data:/ammds/data` | 데이터를 로컬에 저장, 컨테이너를 삭제해도 데이터 유지 |
| `-v $(pwd)/db:/ammds/db` | 데이터베이스를 로컬에 저장, 컨테이너를 삭제해도 데이터 유지 |
| `--restart always` | 컨테이너가 예기치 않게 중지되면 자동 재시작 |
| `qyg2297248353/ammds:latest` | 최신 AMMDS Docker 이미지 사용 |

## 🌐 서비스 접속

배포가 완료되면 브라우저를 열어 AMMDS에 접속할 수 있습니다:

1. 원하는 브라우저를 엽니다
2. `http://localhost:8080`으로 이동 (원격 배포 시 `localhost`를 서버 IP로 변경)
3. AMMDS 로그인 페이지가 표시됩니다

### 📝 기본 계정 정보

| 항목 | 기본값 |
|------------|-------|
| **사용자 이름** | `ammds` |
| **비밀번호** | `ammds` |

### 💡 첫 로그인 시 참고 사항

:::warning
로그인 페이지가 바로 표시되지 않아도 걱정하지 마세요. 시스템이 백그라운드에서 리소스를 다운로드하고 초기화 중일 수 있습니다. 네트워크 속도에 따라 몇 분 정도 걸릴 수 있습니다.
:::

- 로그인 후 **설정에서 즉시 기본 비밀번호를 변경하세요**
- 메인 화면을 둘러보며 AMMDS의 기능에 익숙해지세요
- 오른쪽 아래의 시스템 상태를 확인하여 모든 서비스가 정상 작동하는지 확인하세요
