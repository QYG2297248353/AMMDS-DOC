# 宝塔 패널

宝塔 패널(bt.cn)은 서버 관리 도구로, 복잡한 명령어 조작을 웹 페이지의 버튼과 스위치로 바꿔줍니다. 클릭만으로 웹사이트, 데이터베이스, SSL 인증서 등을 관리할 수 있어 명령어를 외울 필요가 없습니다.

## 준비 사항

시작하기 전에 다음 두 가지가 준비되었는지 확인하세요:

1. **宝塔 패널이 설치되어 있어야 함** — 아직 설치하지 않았다면 [bt.cn](https://bt.cn)에서 설치하세요
2. **宝塔에 Docker가 설치되어 있어야 함** — 宝塔 패널을 열고 왼쪽 소프트웨어 스토어에서 "Docker"를 검색하여 설치하세요

## 宝塔 Docker로 AMMDS 배포하기

### 1단계: Docker 관리 열기

宝塔 패널 왼쪽 메뉴에서 **Docker**를 찾아 클릭하세요.

### 2단계: 컨테이너 생성

1. Docker 관리 페이지에서 **컨테이너** → **컨테이너 생성**을 클릭하세요
2. 다음 정보를 입력하세요:

| 설정 항목 | 입력 내용 | 설명 |
|--------|---------|-----------|
| **이미지 이름** | `qyg2297248353/ammds:latest` | AMMDS의 "설치 패키지" 주소입니다. 입력하면 자동으로 다운로드됩니다 |
| **컨테이너 이름** | `AMMDS` | AMMDS의 이름을 지정하여 쉽게 식별할 수 있습니다 |
| **포트 매핑** | 호스트 포트 `8080` → 컨테이너 포트 `80` | 서버의 8080 포트를 AMMDS에 할당합니다. 접속 시 `http://IP:8080`을 입력하세요 |
| **마운트 디렉터리** | 아래 설명 참고 | 서버의 폴더를 컨테이너에 "매핑"하여, 컨테이너가 재시작되어도 파일이 유지됩니다 |

### 3단계: 마운트 디렉터리 설정 (중요!)

마운트 디렉터리는 서버의 폴더를 AMMDS에 "빌려주는" 것입니다. 최소한 다음 항목을 마운트해야 합니다:

- **데이터 디렉터리**: 서버 폴더(예: `./data`)를 컨테이너의 `/ammds/data`에 마운트
- **데이터베이스 디렉터리**: 서버 폴더(예: `./db`)를 컨테이너의 `/ammds/db`에 마운트
- **다운로드 디렉터리**: 서버 폴더(예: `./download`)를 컨테이너의 `/ammds/download`에 마운트
- **미디어 디렉터리**: 영화/TV 프로그램 폴더(예: `./media`)를 컨테이너의 `/media`에 마운트

:::note
**참고:** 영화 파일이 있는 폴더는 **`/ammds`** 로 시작하는 경로에 마운트하지 마세요. 그렇지 않으면 데이터가 손실될 수 있습니다.

**예시:** 영화가 서버의 `/mnt/movie` 폴더에 있다면, 다음과 같이 마운트하세요: `/mnt/movie:/mnt/movie`
:::

### 4단계: 자동 재시작 설정

컨테이너 생성 페이지에서 **재시작 정책**을 찾아 **"항상 재시작(always)"** 을 선택하세요. 서버가 재부팅되어도 AMMDS가 자동으로 다시 실행됩니다.

### 5단계: 컨테이너 시작

위의 모든 정보를 입력한 후, 확인을 클릭하여 생성하세요. 宝塔이 자동으로 이미지를 가져와 컨테이너를 시작합니다.

### 6단계: AMMDS 접속

브라우저를 열고 다음 주소를 입력하세요:

```
http://서버IP:8080
```

예를 들어 서버 IP가 `192.168.1.100`이라면 `http://192.168.1.100:8080`을 입력하세요.

### 기본 계정 정보

- **사용자 이름**: `ammds`
- **비밀번호**: `ammds`

:::tip
글자가 잘 보이지 않으면 라이트 모드로 전환하세요.
:::

## 주의사항

- **방화벽**: 페이지가 열리지 않으면 서버 방화벽에서 8080 포트를 허용했는지 확인하세요. 宝塔 패널의 보안 페이지에서 포트 허용 규칙을 추가할 수 있습니다
- **데이터 유지**: 마운트 디렉터리 설정은 매우 중요합니다! 설정하지 않으면 컨테이너가 삭제될 때 모든 설정과 데이터가 사라집니다
- **시간대 설정**: 시간대를 변경해야 하는 경우, 컨테이너 생성 시 환경 변수 `TZ=Asia/Shanghai`를 추가하세요
