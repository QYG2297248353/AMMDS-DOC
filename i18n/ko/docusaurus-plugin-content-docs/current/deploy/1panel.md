# 1Panel 배포 가이드

1Panel은 서버 관리 패널로, 웹 화면을 통해 앱, 웹사이트, 파일, 데이터베이스 및 대형 언어 모델(LLM) 등의 리소스를 관리할 수 있습니다. 아래에서는 1Panel에 AMMDS를 배포하는 방법을 설명합니다.

## 타사 앱 스토어 설치

이 방법은 1Panel v1 및 v2 버전에 적용됩니다.

### 자동 설치

자동 설치는 3시간마다 타사 앱 스토어 내용을 자동으로 업데이트하여 항상 최신 앱을 제공합니다.

#### 설치 스크립트

```bash
curl -sSL https://install.lifebus.top/auto_install.sh | bash
```

#### 제거 스크립트

```bash
curl -sSL https://install.lifebus.top/auto_uninstall.sh | bash
```

### 수동 설치

수동 설치는 최신 앱 스토어 내용을 한 번만 가져오며, 이후 업데이트가 필요하면 이 스크립트를 다시 실행해야 합니다.

```bash
curl -sSL https://install.lifebus.top/app_install.sh | bash
```

## 스토어 내용 업데이트

### 작업 단계

1. **스토어 설정 진입**
   왼쪽 탐색 메뉴에서 앱 스토어로 들어가 스토어 설정 페이지로 이동합니다.


2. **업데이트 작업 실행**

   ![스토어 내용 업데이트](/img/deploy/install-1panel-00.png)
   
   #### v1 버전
   스토어 페이지 오른쪽 위의 「앱 목록 업데이트」 버튼을 클릭합니다.

   ![스토어 내용 업데이트](/img/deploy/install-1panel-01.png)

   #### v2 버전
   스토어 페이지 왼쪽 위의 「로컬 앱 동기화」 버튼을 클릭합니다.

## 앱 설치

### 앱 검색 및 선택

![설치](/img/deploy/install-1panel-02.png)

1. **AMMDS 앱 검색**
   앱 스토어에서 "ammds"를 검색합니다.

2. **버전 선택**
   시스템에 두 가지 버전이 표시됩니다:
   - **호환 버전**: 낮은 사양의 장치에 적합
   - **최신 버전**: 높은 사양의 장치에 적합

   ![설치](/img/deploy/install-1panel-03.png)

### 설치 매개변수 설정

설치 버튼을 클릭하면 매개변수 설정 패널이 표시됩니다.

#### 기본 설정

- **이름**: "ammds"로 입력하는 것이 좋지만 필요에 따라 변경 가능
- **버전**: 「최신 버전」을 선택하는 것이 좋음
- **네트워크 모드**: 특별한 요구 사항이 없으면 기본값 유지
- **데이터 지속성 경로**: AMMDS 데이터를 저장할 서버의 절대 경로 입력
- **WebUI 포트**: 8080 권장, 필요에 따라 변경 가능
- **API 포트**: 8081 권장, 필요에 따라 변경 가능

다른 매개변수는 선택 사항으로 필요에 따라 입력하세요.

![설치](/img/deploy/install-1panel-04.png)

#### 마운트 디렉터리 설정

**사용자 정의 마운트 디렉터리**: 미디어 파일이 있는 디렉터리를 입력하는 것이 좋습니다(선택 사항).

![설치](/img/deploy/install-1panel-05.png)

#### 포트 액세스 설정

**「포트 외부 액세스」를 반드시 체크하세요**. 그렇지 않으면 네트워크를 통해 WebUI와 API에 접속할 수 없습니다.

「확인」을 클릭하여 설치를 시작합니다.

## 설치 완료

![설치 완료](/img/deploy/install-1panel-06.png)

### 설치 성공 확인

설치가 완료되면 「설치됨」 탭에서 AMMDS를 확인할 수 있습니다.

### 관리 기능

앱 카드에는 여러 가지 빠른 작업이 제공됩니다:
- **터미널**: 앱 컨테이너 터미널 접속
- **로그**: 실행 로그 확인
- **시작/중지/재시작**: 앱 서비스 관리
- **제거**: 앱 삭제
- **재구성**: 앱 컨테이너 재구축

## 앱 접속

![앱 접속](/img/deploy/install-1panel-07.png)

### 접속 방법

설치가 완료되면 브라우저를 통해 AMMDS의 WebUI와 API에 접속할 수 있습니다:

- **WebUI 주소**: http://&lt;서버IP&gt;:&lt;WebUI포트&gt;
- **API 주소**: http://&lt;서버IP&gt;:&lt;API포트&gt;

### 기본 로그인 정보

- **사용자 이름**: ammds
- **비밀번호**: ammds

:::info 보안 팁
처음 로그인한 후에는 즉시 기본 비밀번호를 변경하여 보안을 유지하세요.
:::

## 고급 시도

### 사용자 정의 설정

Docker에 익숙하다면 1Panel의 고급 설정을 통해 더 세부적인 구성을 할 수 있습니다:

- 리소스 제한 조정(CPU, 메모리)
- 환경 변수 설정
- 네트워크 고급 설정
- 스토리지 볼륨 관리

### 다중 인스턴스 배포

기능 시나리오를 분리해야 하는 경우 1Panel에서 여러 AMMDS 인스턴스를 배포하고 다른 포트와 데이터 경로로 구분할 수 있습니다.

## 경고 및 주의 사항

### 보안 경고

:::danger 중요 보안 안내

- **공식 출처**: 공식 채널에서만 설치 스크립트를 가져오고 타사 출처를 사용하지 마세요
- **비밀번호 보안**: 강력한 비밀번호를 설정하고 정기적으로 변경하세요
- **포트 보안**: 기본 포트를 변경하여 공격 위험을 줄이세요
- **외부 접속**: 필요에 따라 포트 외부 접속을 설정하고 불필요하게 노출하지 마세요

:::

### 시스템 요구 사항

:::info 시스템 구성 요구 사항

#### 최소 구성

- 운영 체제: Linux 배포판(1Panel 설치 지원)
- 프로세서: Intel Core i3 또는 동등 성능
- 메모리: 4GB RAM
- 디스크 공간: 최소 10GB 사용 가능 공간
- 네트워크: 광대역 인터넷 연결

#### 권장 구성

- 운영 체제: Linux 배포판(1Panel 설치 지원)
- 프로세서: Intel Core i5 또는 동등 성능
- 메모리: 8GB RAM 이상
- 디스크 공간: 20GB 사용 가능 공간
- 네트워크: 안정적인 광대역 인터넷 연결

:::

### 사용 주의 사항

- **데이터 백업**: 정기적으로 설정 파일과 중요한 데이터를 백업하세요
- **리소스 관리**: 서버 리소스 사용 상황을 주의하세요
- **네트워크 연결**: 리소스 다운로드 및 업데이트 시 안정적인 네트워크를 유지하세요
- **시스템 유지 관리**: 정기적으로 1Panel과 관련 앱을 업데이트하세요

### 문제 해결

#### 일반적인 문제

- **설치 실패**: 네트워크 연결과 디스크 공간을 확인하세요
- **시작 이상**: 앱 로그에서 오류 정보를 확인하세요
- **접속 실패**: 포트 설정과 방화벽 설정을 확인하세요
- **업그레이드 실패**: 이전 버전을 제거한 후 새 버전을 설치해 보세요

#### 연락처

해결할 수 없는 문제는 공식 GitHub 저장소에 Issue를 제출하세요.

## 모범 사례

### 일상 유지 관리

- **정기 업데이트**: 1Panel과 AMMDS를 최신 버전으로 유지하세요
- **상태 모니터링**: 정기적으로 실행 상태와 리소스 사용 상황을 확인하세요
- **캐시 정리**: 정기적으로 캐시를 정리하여 스토리지 공간을 최적화하세요

### 설정 최적화

- **리소스 할당**: 서버 성능에 따라 리소스를 합리적으로 할당하세요
- **네트워크 최적화**: 적절한 네트워크 설정을 구성하세요
- **스토리지 계획**: 데이터 저장 경로를 합리적으로 계획하세요

### 보안 강화

- **정기 비밀번호 변경**: 정기적으로 로그인 비밀번호를 업데이트하세요
- **접근 제어**: 필요에 따라 접근 제어 정책을 설정하세요
- **로그 감사**: 정기적으로 로그를 확인하여 이상을 발견하세요

### 커뮤니티 참여

- **문제 피드백**: 문제 발생 시及时 피드백하세요
- **경험 공유**: 커뮤니티 토론에 참여하세요
- **코드 기여**: 능력이 있다면 프로젝트 개발에 참여하세요
