# Telegram

Telegram은 클라우드 기반의 인스턴트 메시징 애플리케이션 플랫폼으로，텍스트、음성、비디오、이미지 등 다양한 메시지 형식의 전송과 수신을 지원합니다。이 플러그인은 AMMDS에 Telegram 메시지 푸시 기능을 제공하여 시스템 이벤트의 실시간 알림을 실현합니다。

## 플러그인 설정

![플러그인 설정](/img/plugin/telegram-01.png)

### 기본 설정

| 매개변수 | 설명 | 기본값 |
|------|------|--------|
| 활성 상태 | Telegram 푸시 플러그인 활성화 여부를 제어 | 끔 |
| 서비스 주소 | Telegram API 서비스 엔드포인트 | `https://api.telegram.org` |
| Bot Token | Telegram 봇의 인증 토큰 | - |
| Chat ID | Telegram 대상 채팅의 고유 식별자 | - |
| Webhook URL | (선택사항) Telegram 봇의 Webhook 콜백 주소 | - |
| 스포일러 모드 | 푸시된 이미지에 마스크 처리 추가 여부를 제어 | 끔 |

![플러그인 설정](/img/plugin/telegram-02.png)

### 설정 설명

- **서비스 주소**：기본적으로 공식 API 엔드포인트를 가리킵니다。프록시 또는 자체 서비스를 사용하려면 여기에서 수정할 수 있습니다
- **Bot Token**：Telegram의 `@BotFather` 봇을 통해 얻을 수 있습니다
- **Chat ID**：Telegram의 `@RawDataBot` 봇을 통해 얻을 수 있습니다
- **Webhook URL**：인터넷에 공개된 배포 환경의 사용자만 설정을 추천합니다。비공개 네트워크 환경에서는 메시지 수신에 실패할 수 있습니다
- **스포일러 모드**：활성화하면 푸시된 이미지에 마스크가 추가되어 개인 정보 콘텐츠가 보호됩니다

### 인증 정보 가져오기

1. **Bot Token 가져오기**：
   - Telegram에서 `@BotFather`를 검색하여 시작합니다
   - 가이드에 따라 새 봇을 만들고 Token을 얻습니다

2. **Chat ID 가져오기**：
   - Telegram에서 `@RawDataBot`를 검색하여 시작합니다
   - 임의의 메시지를 보내면 봇이 Chat ID를 포함한 정보를 반환합니다

## 테스트 모듈

### 푸시 테스트

- **기능**：Telegram 푸시 설정이 올바른지 테스트합니다
- **操作**：테스트 버튼을 클릭하면 시스템이 설정된 Chat ID에 테스트 메시지를 보냅니다
- **검증**：성공하면 대상 채팅에서 테스트 메시지를 받게 됩니다

## 기능特性

- **다양한 형식 지원**：텍스트、이미지 등 다양한 메시지 형식을 지원
- **실시간 푸시**：시스템 이벤트를 실시간으로 Telegram으로 푸시
- **스포일러 보호**：이미지 콘텐츠에 마스크 보호 추가 옵션
- **유연한 설정**：사용자 정의 서비스 주소와 Webhook을 지원

## 주의 사항

:::warning Webhook 설정
인터넷에 공개되지 않은 배포 환경의 사용자는 Webhook URL 설정을 추천하지 않습니다。그렇지 않으면 메시지를 수신할 수 없을 수 있습니다。
:::

:::info 서비스 안정성
Telegram 서비스의 네트워크 특성으로 인해 서버 네트워크 환경이 Telegram API에 안정적으로 액세스할 수 있도록 하는 것이 좋습니다。
:::

:::tip 보안 권장 사항
Bot Token은 민감한 정보입니다。미승인된 사람에게 유출되지 않도록 적절히 보관하십시오。
:::
