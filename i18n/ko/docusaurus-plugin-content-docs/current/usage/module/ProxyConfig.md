# 네트워크 프록시 설정

네트워크 프록시 설정은 프로그램이 네트워크 리소스에 액세스할 때 사용할 프록시 서버를 설정하는 데 사용되며, 프록시를 통해서만 액세스할 수 있는 리소스에 프로그램이 정상적으로 액세스할 수 있도록 합니다.

<!-- truncate -->

## 네트워크 프록시 설정에 액세스

다음 두 경로를 통해 네트워크 프록시 설정 페이지에 액세스할 수 있습니다:

1. **작업 관리 >> 네트워크 프록시**
2. **시스템 관리 >> 시스템 설정 >> 네트워크 설정**

## 설정 항목 설명

### 프록시 활성화

**기능 설명**：네트워크 프록시 기능을 활성화할지 여부를 설정합니다.

**작용**：활성화하면 프로그램은 구성된 프록시 서버를 통해 네트워크 리소스에 액세스합니다.

### 프록시 프로토콜

**기능 설명**：네트워크 프록시의 프로토콜 유형을 선택합니다.

**사용 가능한 프로토콜**：
- HTTP
- SOCKS4
- SOCKS5

**추천**：사용 중인 프록시 서버 유형에 따라 적절한 프로토콜을 선택하세요.

### 프록시 호스트

**기능 설명**：네트워크 프록시 서버의 호스트 주소를 입력합니다.

**형식**：IP 주소 (예: `192.168.1.1`) 또는 도메인 이름 (예: `proxy.example.com`)일 수 있습니다.

### 프록시 포트

**기능 설명**：네트워크 프록시 서버의 포트 번호를 입력합니다.

**일반적인 포트**：
- HTTP 프록시: 8080, 3128
- SOCKS 프록시: 1080

### 사용자 이름

**기능 설명**：네트워크 프록시 서버가 인증을 요구하는 경우 인증 사용자 이름을 입력합니다.

### 비밀번호

**기능 설명**：네트워크 프록시 서버가 인증을 요구하는 경우 인증 비밀번호를 입력합니다.

### 프록시 제외 호스트

**기능 설명**：프록시를 통해 액세스할 필요가 없는 호스트 주소를 설정합니다.

**추가 방법**：호스트 주소를 입력한 후 **Enter** 키를 눌러 확인하고 추가합니다.

**적용 시나리오**：
- 로컬 네트워크 리소스
- 프록시를 통해 액세스할 필요가 없는 공공 리소스

## 설정 예

다음은 일반적인 네트워크 프록시 설정 예입니다:

### 예 1: HTTP 프록시

| 설정 항목 | 설정 값 |
|----------|--------|
| 프록시 활성화 | 활성화 |
| 프록시 프로토콜 | HTTP |
| 프록시 호스트 | 192.168.1.100 |
| 프록시 포트 | 8080 |
| 사용자 이름 | user |
| 비밀번호 | password |
| 프록시 제외 호스트 | localhost, 127.0.0.1 |

### 예 2: SOCKS5 프록시

| 설정 항목 | 설정 값 |
|----------|--------|
| 프록시 활성화 | 활성화 |
| 프록시 프로토콜 | SOCKS5 |
| 프록시 호스트 | proxy.example.com |
| 프록시 포트 | 1080 |
| 사용자 이름 | - |
| 비밀번호 | - |
| 프록시 제외 호스트 | localhost, 192.168.1.* |

## 참고 사항

:::warning 중요한 참고 사항
- **구성이 완료된 후 프로그램을 재시작해야 합니다**
- **구성 오류로 인해 프로그램이 정상적으로 실행되지 않을 수 있습니다**
- **프록시 서버 주소와 포트가 올바른지 확인하세요**
- **프록시 서버에 정상적으로 액세스할 수 있는지 확인하세요**
- **인증이 필요한 프록시의 경우 사용자 이름과 비밀번호가 올바른지 확인하세요**
:::

## 자주 묻는 질문

### Q: 왜 네트워크 프록시를 설정해야 하나요?

**A**：일부 네트워크 리소스는 특정 네트워크 환경을 통해서만 액세스할 수 있는 경우가 있으며, 네트워크 프록시를 설정하면 프로그램이 이러한 리소스에 정상적으로 액세스할 수 있습니다. 예를 들어:
- 일부 스크레이퍼는 특정 지역의 네트워크 리소스에 액세스해야 합니다
- 일부 플러그인은 외부 API에 액세스해야 합니다

### Q: 프록시 구성이 올바른지 테스트하려면 어떻게 해야 하나요?

**A**：다음 방법으로 테스트할 수 있습니다:
1. 프록시 설정을 구성합니다
2. 프로그램을 재시작합니다
3. 네트워크 액세스가 필요한 작업 (예: 스크레이핑)을 수행해 보세요
4. 작업이 성공적으로 완료되었는지 확인하세요

### Q: 프록시 설정이 적용된 후 모든 네트워크 요청이 프록시를 통해 전송되나요?

**A**：아니요, 네트워크 액세스가 필요한 특정 작업만 프록시를 통해 전송되며, 프록시 제외 호스트를 설정하여 프록시를 통해 액세스할 필요가 없는 리소스를 지정할 수 있습니다.

### Q: 프록시 제외 호스트를 설정했는데 여전히 프록시를 통해 액세스하고 있나요?

**A**：가능한 이유는 다음과 같습니다:
- 프록시 제외 호스트 형식이 올바르지 않습니다
- 구성이 적용되지 않았습니다 (프로그램 재시작 필요)
- 프로그램의 내부 로직에서 프록시 사용이 우선됩니다

### Q: 프록시 서버 연결 실패는 어떤 영향을 미치나요?

**A**：프록시 서버 연결에 실패하면 다음과 같은 문제가 발생할 수 있습니다:
- 스크레이핑 실패
- 플러그인이 정상적으로 작동하지 않음
- 온라인 리소스를 가져올 수 없음

## 모범 사례

- **안정적인 프록시 서버 사용**：프록시 서버가 안정적이고 신뢰할 수 있는지 확인하여 빈번한 연결 끊김을 방지하세요
- **프록시 제외 호스트 적절히 설정**：로컬 리소스와 프록시가 필요 없는 공공 리소스를 프록시 제외 목록에 추가하여 액세스 속도를 향상시키세요
- **프록시 설정定期 확인**：네트워크 환경이 변경된 경우 프록시 설정을 적시에 업데이트하세요
- **프록시 설정 백업**：설정을 변경하기 전에 현재 설정을 기록하여 문제 발생 시 복원할 수 있도록 하세요
