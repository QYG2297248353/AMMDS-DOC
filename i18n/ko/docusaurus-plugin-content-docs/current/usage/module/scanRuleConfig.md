# 스캔 구성

스캔 구성은 작업 실행 전 시스템이 미디어 파일을 어떻게 인식하고 처리할지 설정하여 스캔 과정이 귀하의 영상 자원을 정확하고 효율적으로 인식할 수 있도록 합니다.

<!-- truncate -->

## 스캔 구성에 액세스

다음 경로를 통해 스캔 규칙 구성 페이지에 액세스할 수 있습니다:

**작업 관리 >> 스캔 규칙 구성**

:::info 작업提示
구성을 업데이트한 후 **저장** 버튼을 클릭해야 합니다. 그렇지 않으면 구성이 적용되지 않습니다.
:::

## 구성 항목 설명

### 형식 제한

**기능 설명**: 시스템이 인식할 수 있는 미디어 파일 유형을 설정합니다.

**기본 지원**: 시스템은 `.mp4`, `.mkv`, `.avi`, `.mov` 등과 같은 일반적인 미디어 파일 유형을 기본적으로 지원합니다.

**사용자 정의 추가**: 필요에 따라 다른 파일 유형을 추가할 수 있습니다. 추가할 때 파일 유형의 확장자(예: `.strm`)를 입력한 후 **Enter** 키를 눌러 입력을 확인하세요.

### 무시 디렉토리

**기능 설명**: 스캔 시 무시해야 하는 디렉토리를 구성합니다. 이러한 디렉토리와 해당 하위 디렉토리의 파일은 스캔되지 않습니다.

**구성 방법**: 디렉토리의 상대 경로(예: `sample`, `tmp`)를 입력합니다. 디렉토리 이름은 대소문자를 구분하지 않습니다.

**적용 시나리오**: 샘플, 임시 파일 또는 기타 비영상 자원을 포함하는 디렉토리를 무시하는 데 사용할 수 있습니다.

### 무시 파일

**기능 설명**: 스캔 시 무시해야 하는 파일을 구성합니다. 지정된 문자열을 포함하는 파일은 스캔되지 않습니다.

**구성 방법**: 파일 이름에 포함된 문자열(예: `sample`, `hh800.com@`)을 입력합니다. 문자열은 대소문자를 구분하지 않습니다.

**적용 시나리오**: 샘플 파일, 특정 표시가 있는 파일 또는 기타 처리할 필요가 없는 파일을 무시하는 데 사용할 수 있습니다.

### 분할 표시

**기능 설명**: 분할된 비디오 파일의 표시 문자열을 구성하여 시스템이 분할된 비디오를 인식하고 개별 파일로 처리하지 않도록 돕습니다.

**구성 방법**: 분할 표시 문자열(예: `part`, `cd`)을 입력합니다. 문자열은 대소문자를 구분하지 않습니다.

**적용 시나리오**: 여러 부분으로 분할된 비디오 파일을 처리하여 시스템이 그들이 동일한 영상 자원에 속하는 것으로 정확하게 인식할 수 있도록 합니다.

### 자막 형식

**기능 설명**: 시스템이 인식할 수 있는 자막 파일 형식을 구성하여 시스템이 정확하게 자막 태그를 추가하도록 돕습니다.


### 최소 비디오 크기

**기능 설명**: 비디오 파일의 최소 크기 임계값을 설정합니다. 스캔 시 이 크기보다 작은 비디오 파일은 무시됩니다.

**단위**: MB

**기본값**: 100MB

**특별한 경우**: `.strm` 등의 파일 유형을 사용하는 경우 최소 비디오 크기를 **0MB**로 설정하는 것이 좋습니다. 그렇지 않으면 이러한 파일을 정상적으로 스캔할 수 없을 수 있습니다.

### 중국어 자막 인식

**기능 설명**: 중국어 자막의 인식 키워드를 구성합니다. 비디오 파일 이름에 이러한 키워드가 포함되어 있을 때 시스템이 자동으로 중국어 자막 태그를 추가합니다.

**예시**: `-C`, `.chs` 등. 키워드는 대소문자를 구분하지 않습니다.

### 모자이크 해제 인식

**기능 설명**: 모자이크 해제의 인식 키워드를 구성합니다. 비디오 파일 이름에 이러한 키워드가 포함되어 있을 때 시스템이 자동으로 모자이크 해제 태그를 추가합니다.

**예시**: `-u`, `.uc` 등. 키워드는 대소문자를 구분하지 않습니다.

### 로컬 nfo 읽기

**기능 설명**: 활성화하면 시스템이 비디오 파일이 있는 디렉토리의 동일한 이름을 가진 `.nfo` 파일을 자동으로 읽고 그 안의 정보를 해석합니다.

**작용**: 해석된 정보는 자동으로 미디어 메타데이터를 생성하여 데이터베이스에 저장되므로 수동으로 메타데이터를 편집하는 작업量을 줄입니다.

## 구성 예제

다음은 전형적인 스캔 구성 예제입니다:

| 구성 항목 | 설정 값 |
|--------|--------|
| 형식 제한 | `.mp4`, `.mkv`, `.avi`, `.mov`, `.strm` |
| 무시 디렉토리 | `sample`, `tmp`, `Extras` |
| 무시 파일 | `sample`, `trailer`, `preview` |
| 분할 표시 | `part`, `cd`, `disc` |
| 자막 형식 | `.srt`, `.ass`, `.idx`, `.sub` |
| 최소 비디오 크기 | 100MB |
| 중국어 자막 인식 | `-C`, `.chs`, `.zh` |
| 모자이크 해제 인식 | `-u`, `.uc`, `uncensored` |
| 로컬 nfo 읽기 | 활성화 |

## 자주 묻는 질문

### Q: 왜 일부 비디오 파일이 스캔되지 않나요?

**A**: 다음과 같은 이유가 있을 수 있습니다:
- 파일 형식이 형식 제한 목록에 없음
- 파일이 있는 디렉토리가 무시 디렉토리에 추가됨
- 파일 이름에 무시된 문자열이 포함됨
- 파일 크기가 설정된 최소 비디오 크기 임계값보다 작음

### Q: 사용자 정의 파일 형식을 정확하게 추가하는 방법은 무엇인가요?

**A**: 사용자 정의 파일 형식을 추가할 때 다음 사항에 주의하세요:
- 파일 확장자의 점을 포함해야 합니다(예: `strm`이 아닌 `.strm`)
- 각 파일 형식을 개별적으로 추가하고 추가 후 Enter 키를 눌러 확인하세요
- 추가하는 파일 형식이 시스템이 처리할 수 있는 유형인지 확인하세요

### Q: 분할 표시의 역할은 무엇인가요?

**A**: 분할 표시는 동일한 영상 자원에 속하는 여러 분할 파일을 시스템이 인식하도록 도와 그들이 여러 독립 파일로 간주되는 대신 하나의 전체로 정확하게 처리되도록 합니다.
