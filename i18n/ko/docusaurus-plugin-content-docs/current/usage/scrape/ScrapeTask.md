# 스크레이핑 작업 실행

스크레이핑 작업은 스크레이핑 프로세스의 두 번째 단계로，스크레이핑 규칙에 따라 스캔 결과 파일을 스크레이핑하여 비디오 메타데이터를获取하고 최종 비디오 파일을 생성하여 비디오 정리 작업을 완료합니다.

<!-- truncate -->

## 전제 조건

스크레이핑 작업을 실행하기 전에 다음 조건이 충족되는지 확인하세요：

### 네트워크 연결

파일을 정리하려면 인터넷을 통해 비디오 메타데이터를获取해야 하므로 프로젝트가 인터넷에 정상적으로 접근할 수 있는지 확인해야 합니다.

### 메타데이터 플러그인

비디오 메타데이터를获取하는 핵심인 관련 메타데이터 플러그인이 활성화되고 정확히 구성되었는지 확인하세요：

+ [Metatube 插件](/docs/plugin/metadata/Metatube)
+ [ThePornDB 插件](/docs/plugin/metadata/ThePornDB)
+ [StashBox 插件](/docs/plugin/metadata/StashBox)
+ [FanzaDMM 插件](/docs/plugin/metadata/FanzaDMM)
+ [Madou 插件](/docs/plugin/metadata/Madou)

:::warning 중요한 경고
관련 메타데이터 플러그인을 활성화하지 않으면 비디오 메타데이터를获取할 수 없어 스크레이핑이失败할 수 있습니다.
:::

:::info 데이터 소스 선언
이 소프트웨어는 어떠한 소스 데이터도 제공하지 않으며，所有 비디오 메타데이터는 사용자가 직접 구성한 플러그인 서비스에서 제공됩니다.
:::

## 스크레이핑 작업 실행

### 단계 1：스크레이핑入库 페이지进入

左侧导航栏을 통해 **작업 관리**에进入한 후 **스크레이핑入库** 옵션을选择하여 스크레이핑入库 페이지에进入하세요.

![스크레이핑入库 페이지](/img/usage/scrape/task-scrape-01.png)

### 단계 2：스크레이핑할 파일 선택

스크레이핑入库 페이지에서 스캔 작업을 통해记录된所有待스크레이핑 파일 목록을 볼 수 있습니다. 필요에 따라 스크레이핑하려는 파일을 선택하세요.

### 단계 3：스크레이핑 작업 실행 시작

페이지의 **스크레이핑整理** 버튼을 클릭하면 시스템이 스크레이핑 작업을 실행하기 시작합니다.

![스크레이핑 작업 실행](/img/usage/scrape/task-scrape-02.png)

### 단계 4：작업 실행 상태查看

작업 실행过程中 페이지에 스크레이핑 작업的执行情况이实时으로展示됩니다，包括：
- 성공적으로 스크레이핑된 파일 수
- 스크레이핑에失败한 파일 수
- 건너뛴 파일 수

### 단계 5：스크레이핑 작업 종료（필요한 경우）

실행过程中 스크레이핑 작업을 종료해야 하는 경우 언제든지 **取消** 버튼을 클릭하여 작업 실행을停止할 수 있습니다.

## 스크레이핑 원리 설명

스크레이핑 작업의 작동 원리는 다음과 같습니다：

1. **스크레이핑 규칙 읽기**：시스템은 당신이 구성한 스크레이핑 규칙에 따라 다른 유형의 파일을 어떻게 처리할지 결정합니다
2. **메타데이터 매칭**：활성화된 메타데이터 플러그인을 통해 비디오의 상세 정보를 매칭하고 획득하려고 시도합니다
3. **비디오 파일生成**：획득한 메타데이터에 따라 시스템이 표준화된 비디오 파일을生成합니다
4. **파일 구조 정리**：사전 설정된 이름 규칙과 디렉토리 구조에 따라 비디오 파일을 정리합니다

## 자주 묻는 질문과 해결方案

### 문제：스크레이핑失败

**可能的原因**：
- 네트워크 연결问题
- 메타데이터 플러그인이 활성화되지 않았거나 구성错误
- 스크레이핑 규칙 설정不当
- 파일命名이 매칭 규칙을满足하지 않음

**해결方案**：
- 네트워크 연결이正常한지检查하세요
- 메타데이터 플러그인이 활성화되고正确히 구성되었는지确认하세요
- 스크레이핑 규칙 설정이合理한지检查하세요
- 파일命名格式을修改하여 매칭 규칙에更符合하도록尝试하세요

### 问题：메타데이터获取不全

**可能的原因**：
- 플러그인 서비스가暂时不可用
- 플러그인 서비스에 비디오 정보가不存在
- 네트워크 지연导致获取 시간 초과

**해결方案**：
- 플러그인 서비스가暂时不可用할 수 있으므로稍后重试하세요
-其他 메타데이터 플러그인을使用하는 것을尝试하세요
- 네트워크 연결稳定性을检查하세요

## 모범 사례

1. **분할 처리**：大量 파일의 경우 시스템负载를过高하게避免하기 위해 분할하여 스크레이핑하는 것이 좋습니다
2. **优先生成**：스크레이핑을执行하기 전에 스크레이핑 규칙和메타데이터 플러그인이正确히 구성되었는지确保하세요
3. **定期 업데이트**：更好的 매칭效果를获取하기 위해 메타데이터 플러그인을定期 업데이트하세요
4. **로그检查**：스크레이핑失败时详细한错误信息를获取하기 위해 시스템 로그를查看하세요

## 주의事项

:::caution 주의
스크레이핑 작업은 파일 수和네트워크 속도에取决于较长时间이 소요될 수 있으므로耐心等待하세요.
:::

:::tip 힌트
최佳 스크레이핑效果를获得하기 위해 파일命名을尽可能规范하게 유지하고 필요한识别信息（번호、제목等）을 포함시키는 것이 좋습니다.
:::
