# 스캔 작업 생성

스캔 작업은 스크레이핑 프로세스의 첫 단계로，스캔 규칙에 따라 대상 디렉토리의 파일을 스캔하여 스캔 결과 파일을 생성합니다. 스캔 작업을 통해 파일을 선택적으로 스크레이핑할 수 있어 스크레이핑 효율을提高할 수 있을 뿐만 아니라 쓰레기 파일에 대한 스크레이핑을 피할 수 있습니다.

<!-- truncate -->

## 작업 목록 접근

다음 경로를 통해 작업 목록 페이지에 접근할 수 있습니다：

**작업 관리 >> 작업 구성 >> 작업 목록**

작업 목록 페이지에서 **작업 생성** 버튼을 클릭하면 스캔 작업을 생성할 수 있습니다.

## 다중 작업 관리

자신의 요구에 따라 여러 개의 다른 스캔 작업을 구성할 수 있습니다. 각 작업은 다른 스캔 디렉토리와 출력 디렉토리를 가질 수 있습니다.

**다중 작업 관리의 장점**：
- 비디오 파일의 분류 및 관리에 도움이 됩니다
- 다른 유형의 비디오 파일에 대해 다른 처리 규칙을 설정할 수 있습니다
- 전체 처리 효율을提高합니다

**예시**：
- 일본 제작 비디오 파일을 전문적으로 스캔하고 지정된 디렉토리에 출력하는 스캔 작업을 생성합니다
- 서양 제작 비디오 파일을 전문적으로 스캔하고 다른 디렉토리에 출력하는 다른 스캔 작업을 생성합니다

## 스캔 작업 생성

![스캔 작업 생성](/img/usage/scrape/task-scan-01.png)

### 작업 이름

**기능 설명**：스캔 작업의 이름을 설정합니다.

**구성 권장 사항**：후속 관리 및 식별을 용이하게 하기 위해 설명적인 이름을 사용하세요. 예："일본 제작 비디오 스캔"、"서양 제작 비디오 스캔" 등.

### 미디어 유형

**기능 설명**：스캔 작업의 미디어 유형을 설정합니다.

**역할**：시스템이 번호를 식별하도록 도와 번호 식별률을 향상시킵니다.

**구성 권장 사항**：혼합 유형의 비디오 파일인 경우 "전체"를 선택하면 됩니다.

### 정리 모드

**기능 설명**：파일 정리의 방식을 설정합니다.

**선택 가능한 모드**：

####原地 정리
- **기능**：정리时 부모 디렉토리에任何操作을 수행하지 않고 원본 파일이 위치한 디렉토리에서直接 정리합니다
- **적용 시나리오**：파일 위치를 변경하지 않고 파일重命名或 메타데이터 정리만希望하는 경우

#### 이동 모드
- **기능**：정리时 파일을 대상 디렉토리로 이동합니다
- **대상 디렉토리**：스크레이핑 구성에서 구성된 형식
- **적용 시나리오**：파일을 지정된 디렉토리에集中 관리하려는 경우

#### 복사 모드
- **기능**：정리时 파일을 대상 디렉토리로 복사합니다
- **대상 디렉토리**：스크레이핑 구성에서 구성된 형식
- **적용 시나리오**：원본 파일을 유지하면서 지정된 디렉토리에 정리된 복사본을 생성하려는 경우

#### 소프트 링크 모드
- **기능**：정리时 파일的 소프트 링크를 대상 디렉토리에 생성합니다
- **대상 디렉토리**：스크레이핑 구성에서 구성된 형식
- **특징**：
  - 추가적인 디스크 공간을 차지하지 않습니다
  - 소프트 링크 파일은 원본 파일을 가리키는 포인터입니다
  - 원본 파일이 삭제되거나 이동된 경우 소프트 링크 파일이 무효화됩니다
- **적용 시나리오**：额外 공간을 차지하지 않으면서 지정된 디렉토리에 파일的 접근入口를 생성하려는 경우

#### 하드 링크 모드
- **기능**：정리时 파일的 하드 링크를 대상 디렉토리에 생성합니다
- **대상 디렉토리**：스크레이핑 구성에서 구성된 형식
- **특징**：
  - 추가적인 디스크 공간을 차지합니다
  - 하드 링크 파일은 원본 파일을 가리키는 포인터입니다
  - 원본 파일이 삭제되거나 이동된 경우 하드 링크 파일이 유효성을 유지합니다
  - 하드 링크 파일은 동일한 파일系统에서만 생성할 수 있습니다
- **적용 시나리오**：동일한 파일系统에서 파일的多个 참조를 생성하여文件安全性을确保하려는 경우
- **주의**：조건이 엄격하여 일반적으로 사용을 권장하지 않습니다

:::info 힌트
原地 정리 모드를 제외한 다른 정리 모드는 대상 디렉토리를 지정해야 합니다.
:::

### 스캔 디렉토리

**기능 설명**：스캔 작업的 대상 디렉토리를 설정합니다.

**역할**：시스템이 해당 디렉토리下的所有 파일을 재귀적으로 스캔합니다.

**구성 권장 사항**：처리가 필요한 비디오 파일이 포함된 디렉토리를选择하세요.

### 정리 디렉토리

**기능 설명**：파일 정리的 대상 디렉토리를 설정합니다.

**적용 시나리오**：原地 정리 모드를 제외한 다른 정리 모드는此 디렉토리를 지정해야 합니다.

**구성 권장 사항**：정리后的文件을 저장하기 위해 충분한 저장 공간이 있는 디렉토리를选择하세요.

###重命名 규칙

**기능 설명**：파일重命名的 방식을 설정합니다.

**선택 가능한 규칙**：

#### 원본 파일 이름
- **기능**：정리时 원본 파일 이름을不变으로 유지합니다
- **적용 시나리오**：파일的 원본命名을保留하려는 경우

#### 표준화된 파일 이름
- **기능**：정리时 스크레이핑 구성中 구성된 파일 이름格式에 따라 파일 이름을 표준화합니다
- **命名依据**：스크레이핑 구성中 구성된 파일 이름格式에 따라 결정됩니다
- **적용 시나리오**：파일命名格式을统一하여文件管理的规范性을提高하려는 경우

### 중복 파일 처리

**기능 설명**：대상 파일이已存在하는情况的 처리 방식을 설정합니다.

**구성 권장 사항**：스킵、덮어쓰기、重命名等 필요에 따라合适的 처리 방식을选择하세요.

### 디렉토리 모니터링

**기능 설명**：활성화后 최초 초기화 스캔 작업后 시스템이 이 디렉토리를 모니터링하여 새로운 파일이出现时 자동으로 스캔并 스캔结果文件에添加합니다.

**적용 시나리오**：시스템이 새로 추가된 파일을 자동으로 처리하도록希望하며手动으로 스캔을触发할 필요가 없는 경우

### 즐겨찾기 추가

**기능 설명**：활성화后 정리后此 비디오 메타데이터를自动으로 즐겨찾기에 추가합니다.

**적용 시나리오**：重要한 비디오 자원을快速으로标记和访问하려는 경우

### 계획 작업

**기능 설명**：스캔 작업的自动执行时间을 설정합니다.

**주의**：此 항목은 디렉토리 모니터링과冲突하므로 둘 중 하나만选择할 수 있습니다.

**구성格式**：Spring 스타일的 cron 표현식을使用합니다

**예시**：每小时执行一次 스캔 작업的 경우 `0 0 * * * ?`를填写합니다

## 스캔 작업执行

생성 완료 후 **즉시 스캔** 버튼을 클릭하면 스캔 작업을执行할 수 있습니다.

![스캔 작업执行](/img/usage/scrape/task-scan-02.png)

### 작업 관리操作

- **编辑**：**编辑** 버튼을 클릭하면 스캔 작업的配置를修改할 수 있습니다
- **记录**：**记录** 버튼을 클릭하면 스캔 작업的执行历史记录를查看할 수 있습니다
- **删除**：**删除** 버튼을 클릭하면不需要的 스캔 작업을删除할 수 있습니다

### 스캔 결과查看

![스캔 결과查看](/img/usage/scrape/task-scan-03.png)

스캔结束后新增的 스크레이핑 가능한 파일은 **작업 관리 >> 스크레이핑入库**에서查看和处理하세요.

## 계획 작업

프로젝트는 계획 작업의 실행 시간을 지정하기 위해 Spring 스타일的 cron 표현식을 사용합니다.

###常用 표현식

| 표현식 | 설명 |
|--------|------|
| `0 0 * * * ?` |每小时执行一次 |
| `0 0 0 * * ?` |每天凌晨执行一次 |
| `0 0 0 * * 1` |每周一凌晨执行一次 |
| `0 0 0 1 * ?` |每月1日凌晨执行一次 |

## 자주 묻는 질문

### Q: 스캔 작업执行失败怎么办？

**A**：可能的原因包括：
- 스캔 디렉토리가不存在하거나访问 권한이 없음
- 정리 디렉토리가不存在하거나写入 권한이 없음
- 구성 오류
- 시스템 리소스不足

**해결方案**：
- 디렉토리 경로和权限를检查하세요
- 구성参数를检查하세요
- 시스템에 충분한 리소스가 있는지 확인하세요
-具体的错误信息를了解하기 위해执行记录를查看하세요

### Q: 디렉토리 모니터링和계획 작업的区别是什么？

**A**：
- **디렉토리 모니터링**：디렉토리变化를实时으로监控하여新文件添加时立即处理
- **계획 작업**：预设的时间间隔에 따라定期적으로 스캔을执行

### Q: 소프트 링크和하드 링크的区别是什么？

**A**：
- **소프트 링크**：Windows的 바로가기와类似，额外空间을 차지하지 않지만 원본 파일에依赖
- **하드 링크**：파일的多个引用와类似，额外空间을 차지하지만 원본 파일에依赖하지 않음（원본 파일删除后仍可使用）

### Q:为什么 스캔结果中某些文件이显示되지 않습니까？

**A**：可能的原因包括：
- 파일格式이 스캔 규칙的格式限制 목록에 없음
- 파일이 위치한 디렉토리가忽略 디렉토리에添加됨
- 파일 이름에忽略된 문자열이 포함됨
- 파일 크기가设置的最小 비디오 크기阈值보다 작음

### Q: 스캔 효율을提高하려면 어떻게 해야 합니까？

**A**：다음 방법을 시도할 수 있습니다：
-忽略 디렉토리和忽略文件을合理하게设置하세요
-不同类型的文件에 대해专门的 스캔 작업을创建하세요
-重复 스캔을避免하기 위해 디렉토리 모니터링 기능을使用하세요
-시스템 리소스分配을适当히调整하세요

## 모범 사례

- **작업分类**：비디오 유형에 따라不同的 스캔 작업을创建하여处理效率를提高하세요
- **合理命名**：후속管理을 용이하게 하기 위해 설명적인 작업 이름을使用하세요
- **定期维护**：不需要的 스캔 작업을定期적으로检查和清理하세요
- **配置 백업**：问题出现时恢复할 수 있도록重要한配置를修改하기前 현재配置를记录하세요
- **执行监控**：문제를及时发现和解决하기 위해 스캔 작업的执行记录를定期적으로查看하세요
