# Windows 배포 가이드

Microsoft Windows（간단히 Windows）는 마이크로소프트사가 개발한 일련의 독점 상용 운영 체제로，그래픽 사용자 인터페이스를 핵심 특성으로，개인 컴퓨터，서버 등의 기기에서 널리 사용됩니다. 본 가이드는 Windows 환경에서 AMMDS를 배포하는 과정을 자세히 설명합니다.

<!-- truncate -->

## 다운로드

![AMMDS Launcher 다운로드 페이지](/img/deploy/install-windows-00.png)

### 공식 배포 주소

[AMMDS Launcher GitHub Releases](https://github.com/QYG2297248353/AMMDS-Launcher/releases)

### 사용 가능한 설치 패키지 유형

| 파일 이름            | 유형     | 설명                                                                 |
| ----------------- | -------- | -------------------------------------------------------------------- |
| `ammds-setup.exe` | 설치 프로그램 | 완전한 AMMDS Launcher 설치 패키지，자동 설치 마법사와 모든 필요한 구성 요소를 포함합니다. |
| `ammds.zip`       | 휴대용 버전   | 그린 설치 필요 없는 버전，압축을 푼 후 바로 실행할 수 있으며，임시 사용 또는 관리자 권한이 없는 환경에 적합합니다. |

### 권장 다운로드

**권장 다운로드：** `ammds-setup.exe`

설치 프로그램을 사용하면 모든 종속 구성 요소가 올바르게 설치되고，보다 나은 시스템 통합 경험을 얻을 수 있습니다.

## 설치

### 전제 조건

- 운영 체제：Windows 10 또는 상위 버전
- 권한：설치를 위해 관리자 권한이 필요합니다.
- 네트워크：최초 실행 시必要한 리소스를 다운로드하기 위해 네트워크에 연결되어 있어야 합니다.

### 설치 단계

1. **설치 프로그램 실행**

   다운로드한 `ammds-setup.exe` 파일을 더블 클릭하여 설치 마법사를 시작합니다.

2. **설치 언어 선택**

   ![설치 언어 선택 인터페이스](/img/deploy/install-windows-01.png)

   언어 선택 인터페이스에서 **간체 중국어**를 설치 언어로 선택한 다음，「확인」을 클릭합니다.

3. **추가 구성 요소 선택**

   ![추가 구성 요소 선택 인터페이스](/img/deploy/install-windows-02.png)

   구성 요소 선택 인터페이스에서 시스템은 사용 가능한 모든 추가 구성 요소를 나열합니다. 완전한 기능을 얻기 위해 **모든 구성 요소를 선택**하는 것이 좋습니다：
   - 핵심 구성 요소：AMMDS Launcher 주 프로그램
   - 종속 라이브러리：필요한 런타임 환경
   - 시스템 통합：트레이 아이콘 및 자동 시작 기능

4. **설치 시작**

   ![설치 확인 인터페이스](/img/deploy/install-windows-03.png)

   설치 구성을 확인한 후，「설치」버튼을 클릭하여 설치 과정을 시작합니다.

5. **설치 진행 상황**

   ![설치 진행 상황 인터페이스](/img/deploy/install-windows-04.png)

   설치 프로그램은 현재 설치 진행 상황과 상태를 표시합니다. 설치가 완료될 때까지 기다려 주십시오.

6. **설치 완료**

   ![설치 완료 인터페이스](/img/deploy/install-windows-05.png)

   설치가 완료되면，「완료」버튼을 클릭하여 설치 마법사를 종료합니다.

### 최초 실행

설치가 완료되면，시스템은 자동으로 AMMDS Launcher를 시작하고 공식 문서 홈페이지를 표시합니다.

![최초 실행 흰 화면 상태](/img/deploy/install-windows-06.png)

**참고：** 최초 실행 시 흰 화면 현상이 나타나면，이는 정상적인 초기화 과정입니다. 백그라운드에서 필요한 리소스 파일을 다운로드하고 있습니다. 기다려 주십시오. 다운로드가 완료되면 페이지가 자동으로 새로 고침됩니다.

![AMMDS Launcher 주 인터페이스](/img/deploy/install-windows-07.png)

리소스 다운로드가 완료되면，시스템은 자동으로 AMMDS Launcher 주 인터페이스에 진입합니다. 이제 모든 기능을 사용할 수 있습니다.

### 설치 주의 사항

- 설치 과정 중 설치 창을 닫지 마십시오.
- 최초 실행 시 네트워크 연결이 정상적인지 확인하십시오.
- 설치 실패가 발생하면，충분한 디스크 공간과 관리자 권한이 있는지 확인하십시오.

## 업그레이드

### 업그레이드 전 준비

- **데이터 백업**：업그레이드 전 AMMDS의 구성 파일과 중요한 데이터를 백업하는 것이 좋습니다.
- **프로그램 닫기**：AMMDS Launcher가 완전히 닫혀 있는지 확인하십시오. 트레이 아이콘을 포함합니다.
- **새 버전 다운로드**：공식 GitHub Releases 페이지에서 최신 버전의 `ammds-setup.exe`를 다운로드합니다.

### 업그레이드 단계

1. **업그레이드 프로그램 실행**

   다운로드한 새 버전 `ammds-setup.exe` 파일을 더블 클릭하여 업그레이드 마법사를 시작합니다.

2. **업그레이드 마법사 따르기**

   업그레이드 마법사는 자동으로 설치된 버전을 감지하고 덮어쓰기 업그레이드를 제안합니다. 마법사의 지침에 따라 업그레이드 과정을 완료합니다.

3. **업그레이드 완료**

   업그레이드가 완료되면，「완료」버튼을 클릭하여 마법사를 종료합니다. 시스템은 자동으로 새 버전 AMMDS Launcher를 시작합니다.

### 업그레이드 메커니즘 설명

AMMDS Launcher는 **덮어쓰기 업그레이드** 메커니즘을 채택합니다. 구체적인 특징은 다음과 같습니다：

- 사용자 구성 및 데이터 보존：업그레이드 과정에서 기존 구성 파일과 데이터 디렉토리가 보존됩니다.
- 핵심 구성 요소 업데이트：이전 버전의 실행 파일과 종속 라이브러리가 자동으로 교체됩니다.
- 시스템 통합 유지：트레이 아이콘，자동 시작 등의 시스템 통합 설정이 보존됩니다.

### 업그레이드 후 검증

업그레이드가 완료된 후，다음과 같은 검증을 권장합니다：

1. AMMDS Launcher가 정상적으로 시작되는지 확인합니다.
2. 버전 번호가 업데이트되었는지 확인합니다（「정보」페이지에서 확인할 수 있습니다）.
3. 모든 기능이 정상적으로 작동하는지 확인합니다.
4. 데이터와 구성이 완전히 보존되었는지 확인합니다.

### 업그레이드 주의 사항

- 업그레이드 과정 중 업그레이드 창을 닫지 마십시오.
- 업그레이드 실패가 발생하면，이전 버전을 먼저 제거한 후 새 버전을 설치해 보십시오.
- 주요 버전 업그레이드 시 일부 리소스 파일을 다시 다운로드해야 할 수 있습니다.

## 시스템 트레이 관리

![AMMDS Launcher 트레이 메뉴](/img/deploy/install-windows-08.png)

### 트레이 기능 개요

AMMDS Launcher가 설치되면，Windows 시스템 트레이에 전용 아이콘이 생성되어 빠른 액세스 및 관리 기능을 제공합니다. 트레이 아이콘을 마우스 오른쪽 버튼으로 클릭하면 기능 메뉴가 열리고 다양한 작업을 수행할 수 있습니다.

### 트레이 메뉴 기능

| 메뉴 항목           | 기능 설명                   | 기술 설명                                                  |
| ---------------- | -------------------------- | --------------------------------------------------------- |
| **데이터 디렉터리 열기** | AMMDS의 리소스 저장 디렉터리를 엽니다.    | `%APPDATA%\AMMDS` 디렉터리로 직접 이동합니다. 데이터，구성 및 로그 파일을 포함합니다. |
| **시작**         | AMMDS Launcher 서비스를 시작합니다.     | 백그라운드 서비스 프로세스를 시작하여 연결을 대기합니다.                            |
| **재시작**         | AMMDS Launcher를 재시작합니다.         | 기존 프로세스를 먼저 중지한 후 다시 시작합니다. 구성 변경 사항을 적용하는 데 사용됩니다.              |
| **중지**         | AMMDS Launcher 서비스를 중지합니다.     | 백그라운드 서비스 프로세스를 중지하여 시스템 리소스를 해제합니다.                            |
| **종료**         | AMMDS Launcher를 완전히 종료합니다.     | 서비스를 중지하고 트레이 아이콘을 제거하여 모든 리소스를 완전히 해제합니다.                  |
| **패널 열기**     | AMMDS Launcher 주 인터페이스를 엽니다.   | Web 관리 인터페이스를 시작하고 표시합니다. AMMDS를 구성하고 관리하는 데 사용됩니다.                |
| **자동 시작 활성화** | 시스템 시작 시 자동 시작 기능을 전환합니다. | Windows 시작 항목에 AMMDS Launcher를 추가하거나 제거합니다.                 |

### 트레이 아이콘 상태 설명

- **컬러 아이콘**：AMMDS Launcher 서비스가 실행 중입니다.
- **회색 아이콘**：AMMDS Launcher 서비스가 중지되었습니다.
- **깜박이는 아이콘**：AMMDS Launcher가 시작 또는 재시작 과정 중입니다.

### 사용 권장 사항

- **일상 관리**：트레이 메뉴를 사용하여 빠른 시작，중지 및 재시작 작업을 수행합니다.
- **문제 해결**：문제가 발생할 때，재시작 기능을 사용하여 해결해 보십시오.
- **시스템 최적화**：AMMDS를 장기간 사용하지 않는 경우，모든 리소스를 완전히 해제하기 위해 종료 기능을 사용하는 것이 좋습니다.
- **자동 시작**：지속적으로 실행해야 하는 시나리오의 경우，자동 시작 기능을 활성화하는 것이 좋습니다.

## 실행 로그 및 데이터 관리

### 데이터 디렉터리 구조

![AMMDS 데이터 디렉터리 구조](/img/deploy/install-windows-09.png)

AMMDS의 모든 데이터와 구성 파일은 전용 데이터 디렉터리에 저장됩니다. 기본 경로는：`%APPDATA%\AMMDS`

**디렉터리 구조 설명：**

| 디렉터리   | 기능 설명              | 기술 세부 사항                                                   |
| ------ | --------------------- | ---------------------------------------------------------- |
| `data` | AMMDS 리소스 파일 저장     | 모델 파일，구성 템플릿，사전 설정 등 핵심 리소스를 포함합니다. 최초 실행 시 자동으로 다운로드됩니다. |
| `db`   | AMMDS 데이터베이스 파일 저장   | 사용자 구성，실행 기록，작업 기록 등 구조화된 데이터를 포함합니다.               |
| `logs` | AMMDS 실행 로그 파일 저장 | 시스템 로그，오류 기록，리소스 다운로드 로그 등을 포함합니다.                     |

### 로그 파일 설명

![AMMDS 리소스 다운로드 로그](/img/deploy/install-windows-10.png)

#### 로그 파일 저장 구조

AMMDS의 로그 파일은 주로 두 위치에 저장됩니다：

**1. AMMDS 핵심 로그（data/logs 디렉터리）**
| 로그 파일 | 기능 설명 |
|---------|----------|
| `data/logs/ammds.log` | 시스템 실행 로그，AMMDS의 시작，중지，오류 등 시스템 이벤트를 기록합니다. |
| `data/logs/ammds-202x-xx-xx.log` | 날짜로命名된 실행 로그，AMMDS의 자세한 실행 상태와操作记录를 기록합니다. |
| `data/logs/ammds-error.log` | 오류 로그，시스템 실행 과정中的 오류 정보를专门记录합니다. |

**2. Windows 시작기 로그（logs 디렉터리）**
| 로그 파일 | 기능 설명 |
|---------|----------|
| `logs/launcher.log` | 시작기 로그，Windows AMMDS Launcher의 시작 과정，구성 정보和系统集成状态를记录합니다. |

#### 리소스 다운로드 로그

**리소스 다운로드 로그**（관련 로그 파일에 포함）：

- 리소스 파일 이름 및 버전 정보
- 다운로드 진행 상황 및 속도 통계
- 다운로드 상태（성공/실패/재시도）
- 다운로드 타임스탬프 및 소요 시간
- 다운로드 서버 정보

:::info 진행 상황 표시
로그에 진행 상황이 100%에 도달했다는 메시지가 표시되면，필요한 모든 리소스 파일이 다운로드 완료된 것을 의미합니다.
:::

### 로그 사용 방법

#### 1. 문제 해결

**AMMDS 핵심 문제**

- `data/logs/ammds-error.log`를查看하여 자세한 오류 정보를获取합니다.
- `data/logs/ammds.log`를参考하여系统启动和初始化状态를了解합니다.
- `data/logs/ammds-202x-xx-xx.log`를分析하여错误发生前后的操作记录를查看합니다.

**시작기 문제**

- `logs/launcher.log`를查看하여 시작기启动 과정和配置状态를了解합니다.
- AMMDS 핵심과의通信状态를检查합니다.

#### 2. 리소스 관리

**리소스 다운로드 문제**

- 로그中的 다운로드记录를分析하여，다운로드 실패한 리소스를排查합니다.
- 다운로드 서버 연결状态를检查합니다.
- 리소스文件版本和完整性를确认합니다.

**디스크 공간 관리**

- 로그文件大小를监控하고，定期적으로过期된 로그를清理합니다.
- 리소스文件 다운로드量를分析하여，합리적으로 디스크 공간을规划합니다.

#### 3. 성능分析

**시스템 성능**

- 로그中的 타임스탬프를分析하여，각 작업의 실행 시간을了解합니다.
- 리소스使用情况를监控하여，시스템 구성를优化합니다.
- 성능 병목 현상을识别하여，시스템 매개변수를调整합니다.

**시작 시간 최적화**

- `logs/launcher.log`中的启动 과정 소요时间를分析합니다.
- `data/logs/ammds.log`中的 초기화时间를检查합니다.
- 시작 항목和구성을优化하여，시작 시간를减少합니다.

#### 4. 로그文件管理

**로그 회전**

- AMMDS는 자동으로 로그 회전을 수행하여 날짜로命名된 로그文件를生成합니다.
-过期된 로그는压缩或清理되어合理한 디스크 점유율을 유지합니다.

**로그 백업**

-重要한 로그文件를定期적으로 백업합니다. 특히 오류 로그입니다.
-시스템 업그레이드前 현재 로그를 백업하여，문제 비교 분석에 용이합니다.

**로그级别调整**

-필요에 따라 로그级别를调整하여，详细程度和性能影响를平衡합니다.
-문제 해결时详细 로그를使用하고，正常运行时标准 로그를使用합니다.

### 최초 실행 주의 사항

:::warning 최초 실행 안내
최초로 AMMDS를 실행할 때，시스템은 다음 작업을 수행합니다. 기다려 주십시오：

- **리소스 다운로드**：모델 파일，종속 라이브러리 등 필요한 리소스 파일을 자동으로 다운로드합니다.
- **시간 예상**：네트워크 속도에 따라，최초 실행에는 5-30분 정도 소요될 수 있습니다.
- **네트워크 요구 사항**：네트워크 연결이 안정적인지 확인하십시오. 다운로드 과정中 네트워크를中断하지 마십시오.
- **디스크 공간**：다운로드된 리소스 파일을 저장하기 위해 충분한 디스크 공간이 필요합니다. 최소 10GB 공간을 예약하는 것이 좋습니다.
  :::

## 경고 및 주의 사항

### 보안 경고

:::danger 중요 보안 안내

- **공식 출처**：공식 GitHub Releases 페이지에서만 AMMDS 설치 패키지를 다운로드하십시오. 타사 다운로드 원본을 사용하지 마십시오.
- **안티바이러스 소프트웨어**：설치 과정中 안티바이러스 소프트웨어 경고가 나타나면，오탐으로 확인 후 신뢰 목록에 추가하십시오.
- **관리자 권한**：설치和 업그레이드 과정需要管理员权限，请确保以管理员身份运行安装程序.
  :::

### 시스템 요구 사항

:::info 시스템 구성 요구 사항

#### 최소 구성

- 운영 체제：Windows 10 64비트
- 프로세서：Intel Core i3 또는 동등 성능
- 메모리：4GB RAM
- 디스크 공간：최소 10GB 사용 가능 공간
- 네트워크：광대역 인터넷 연결

#### 권장 구성

- 운영 체제：Windows 10/11 64비트
- 프로세서：Intel Core i5 또는 동등 성능
- 메모리：8GB RAM 또는 그 이상
- 디스크 공간：20GB 사용 가능 공간
- 네트워크：안정적인 광대역 인터넷 연결

:::

### 사용 주의 사항

- **데이터 백업**：AMMDS의 데이터 디렉터리를定期적으로 백업합니다. 특히重要한 구성文件和模型文件입니다.
- **리소스 관리**：디스크 공간 사용情况을 모니터링하고，필요하지 않은 리소스文件을 적시에清理하십시오.
- **네트워크 연결**：리소스를 다운로드하고 업데이트할 때 특히 안정적인 네트워크 연결을确保하십시오.
- **시스템 유지 관리**：Windows 시스템和相关驱动程序을定期적으로 업데이트하여，시스템 안정성을确保하십시오.

### 문제 해결

#### 일반적인 문제

- **설치 실패**：관리자 권한和디스크 공간을检查하십시오.
- **시작 흰 화면**：리소스 다운로드가 완료될 때까지 기다리거나，네트워크 연결을检查하십시오.
- **기능 이상**：로그文件를查看하여问题를定位하거나，AMMDS Launcher를 재시작하십시오.
- **업그레이드 실패**：이전 버전을 먼저 제거한 후 새 버전을 설치해 보십시오.

#### 연락처

해결할 수 없는 문제가 발생하면，공식 GitHub 저장소에 Issue를 제출하거나 도움을 요청하십시오.
