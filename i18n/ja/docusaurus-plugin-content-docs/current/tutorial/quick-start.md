# クイックスタート

:::info
AMMDS へようこそ！このガイドでは、Windows、飛牛 Nas、または Docker などのサポートされている方法の 1 つを使用して AMMDS をデプロイするのに役立ちます。
:::

## 🚀 デプロイ方法

### Windows デプロイ

:::note
**パッケージタイプ**：EXE インストーラー
:::

Windows ユーザー向けに、シンプルで直感的な実行可能インストーラーを提供しています：

1. [AMMDS Launcher リリースページ](https://github.com/QYG2297248353/AMMDS-Launcher/releases) から最新の `ammds-setup.exe` をダウンロードします
2. ダウンロードした `ammds-setup.exe` ファイルを見つけます
3. 右クリックして **管理者として実行** を選択します
4. 直感的な画面ウィザードに従ってインストールを完了します
5. インストールが完了すると、サービスが自動的に起動します

---

### 飛牛 Nas デプロイ

:::note
**パッケージタイプ**：FPK（飛牛パッケージ）
:::

飛牛 Nas ユーザー向けに、飛牛アプリストアに直接インポートするために特別に設計された FPK パッケージを提供しています：

1. [飛牛 Nas リリースページ](https://github.com/QYG2297248353/AMMDS-Offline-FnNas/releases) から最新の `AMMDS.offline.fpk` をダウンロードします
2. 飛牛 Nas ダッシュボードにアクセスします
3. **飛牛アプリストア** を開きます
4. **ローカルパッケージのインポート** を選択し、ダウンロードした FPK ファイルを見つけます
5. 指示に従ってインストールを完了します
6. サービスは飛牛 Nas アプリストアインターフェースを介して管理されます

---

### Docker デプロイ

:::note
**パッケージタイプ**：Docker イメージ
:::

Docker がインストールされているユーザー向けに、単一のコマンドで AMMDS をデプロイします：

```bash showLineNumbers title="Docker Command"
docker run -itd \
   --name AMMDS \
   -p 8080:80 \
   -v $(pwd)/data:/ammds/data \
   -v $(pwd)/db:/ammds/db \
   --restart always \
   qyg2297248353/ammds:latest
```

#### Docker コマンドの説明

| オプション | 説明 |
|--------|-------------|
| `-itd` | コンテナをインタラクティブモードで実行し、疑似 TTY を使用し、ターミナルから切り離します |
| `--name AMMDS` | コンテナに "AMMDS" という名前を割り当て、管理を容易にします |
| `-p 8080:80` | ホストのポート 8080 をコンテナのポート 80（Web インターフェース）にマッピングします |
| `-v $(pwd)/data:/ammds/data` | アプリケーションデータの永続ストレージに使用します |
| `-v $(pwd)/db:/ammds/db` | データベースファイルの永続ストレージに使用します |
| `--restart always` | コンテナが予期せず停止した場合に、コンテナを自動的に再起動します |
| `qyg2297248353/ammds:latest` | 最新の AMMDS Docker イメージを使用します |

## 🌐 サービスへのアクセス

デプロイに成功した後、Web ブラウザを使用して AMMDS にアクセスします：

1. お気に入りの Web ブラウザを開きます
2. `http://localhost:8080` に移動します（リモートデプロイの場合は、`localhost` をサーバーの IP に置き換えます）
3. AMMDS ログインページが表示されます

### 📝 デフォルトの認証情報

| 認証情報 | 値 |
|------------|-------|
| **ユーザー名** | `ammds` |
| **パスワード** | `ammds` |

### 💡 初回アクセスのヒント

:::warning
すぐにログインページが表示されない場合は、しばらくお待ちください。システムはまだバックグラウンドでリソースをダウンロードして初期化している可能性があります。ネットワーク速度によっては、このプロセスに数分かかる場合があります。
:::

- ログイン後、**設定メニューからすぐにデフォルトパスワードを変更してください**
- 直感的なダッシュボードを探索し、AMMDS の機能に慣れましょう
- 右下隅のシステムステータスを確認し、すべてのサービスが実行されていることを確認します
