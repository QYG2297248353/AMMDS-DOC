# スクレイピングタスクの実行

スクレイピングタスクは、動画整理の第 2 ステップです。簡単に言うと、システムがスクレイピングルールに従ってネット上から動画の情報（表紙、あらすじ、出演者など）を探し、整理して指定の場所に保存します。

> **「スクレイピング」って何？** スクレイピングとは、システムが自動的にネット上から動画の情報（表紙画像、ストーリー概要、キャスト、評価など、まとめて「メタデータ」と呼びます）を収集し、ダウンロードしてあなたの動画ファイルとペアリングすることです。動画ファイルに「ラベルを貼る」ようなものです。

<!-- truncate -->

## 前提条件

スクレイピングタスクを実行する前に、以下の準備が整っていることを確認してください：

### ネットワーク接続

ファイルの整理にはインターネット経由で動画情報を取得する必要があるため、ネットワークが通じていることを確認してください。

### メタデータプラグイン

関連するメタデータプラグインが有効になっており、正しく設定されていることを確認してください。これらのプラグインは動画情報を見つけるための重要なツールです：

+ [Metatube プラグイン](/docs/plugin/metadata/Metatube)
+ [ThePornDB プラグイン](/docs/plugin/metadata/ThePornDB)
+ [StashBox プラグイン](/docs/plugin/metadata/ThePornDB)
+ [FanzaDMM プラグイン](/docs/plugin/metadata/FanzaDmm)
+ [Madou プラグイン](/docs/plugin/metadata/Madou)

:::warning 重要な注意事項
メタデータプラグインが有効になっていないと、動画情報を取得できず、スクレイピングは必ず失敗します。
:::

:::info データソースの声明
本ソフトウェアは一切のソースデータを提供しません。すべての動画情報はユーザー自身が設定したプラグインサービスから取得されます。
:::

## スクレイピングタスクの実行

### ステップ 1：スクレイピング入庫ページに入る

左側のナビゲーションバーから **タスク管理** に入り、**スクレイピング入庫** を選択します。

![スクレイピング入庫ページ](/img/usage/scrape/task-scrape-01.png)

### ステップ 2：スクレイピングするファイルを選択

スクレイピング入庫ページでは、すべてのスクレイピング待ちファイルのリストが表示されます。処理したいファイルをチェックしてください。

### ステップ 3：スクレイピングを開始

**スクレイピング整理** ボタンをクリックすると、システムが処理を開始します。

![スクレイピングタスクの実行](/img/usage/scrape/task-scrape-02.png)

### ステップ 4：タスクの状態を確認

タスク実行中、ページにリアルタイムで進捗が表示されます：
- スクレイピング成功ファイル数
- スクレイピング失敗ファイル数
- スキップされたファイル数

### ステップ 5：タスクを中止する（必要な場合）

実行中に中止したい場合は、いつでも **キャンセル** ボタンをクリックできます。

## スクレイピングの仕組み

スクレイピングの全体的な流れは次のとおりです：

1. **スクレイピングルールの読み取り**：システムは設定されたルールに従って、ファイルの処理方法を決定します
2. **メタデータのマッチング**：有効化されたメタデータプラグインを通じて、動画の詳細情報を検索します
3. **動画ファイルの生成**：取得した情報に基づいて、標準化された動画ファイルを生成します
4. **ファイル構造の整理**：事前設定された命名ルールとディレクトリ構造に従ってファイルを整理します

## よくある問題と解決策

### 問題：スクレイピングに失敗する

**考えられる原因**：
- ネットワークに接続できない
- メタデータプラグインが無効か設定ミス
- スクレイピングルールの設定が不適切
- ファイル名が認識ルールに合致しない

**解決方法**：
- ネットワークが正常か確認
- メタデータプラグインが有効で設定が正しいことを確認
- スクレイピングルールの設定が適切か確認
- ファイル名を変更して認識されやすくしてみる

### 問題：動画情報が不完全

**考えられる原因**：
- プラグインサービスが一時的に利用不可
- その動画がプラグインサービスに存在しない
- ネットワークが遅くて取得がタイムアウト

**解決方法**：
- 時間をおいて再試行（一時的な問題の可能性）
- 別のメタデータプラグインを試す
- ネットワークの安定性を確認

## ベストプラクティス

1. **分割処理**：ファイルが多い場合は分割してスクレイピングすることを推奨。一度に大量に処理するとシステムが重くなる可能性があります
2. **事前設定の確認**：スクレイピング前に、スクレイピングルールとメタデータプラグインが正しく設定されていることを確認
3. **プラグインの定期更新**：プラグインを更新するとマッチング精度が向上
4. **ログでエラーチェック**：スクレイピングに失敗したら、システムログで詳細なエラー情報を確認

## 注意事項

:::caution 注意
スクレイピングタスクは、ファイル数やネットワーク速度によっては完了までに時間がかかる場合があります。気長にお待ちください。
:::

:::tip ヒント
スクレイピングの精度を高めるには、ファイル名をできるだけ統一し、必要な識別情報（作品番号、タイトルなど）を含めることをおすすめします。
:::
