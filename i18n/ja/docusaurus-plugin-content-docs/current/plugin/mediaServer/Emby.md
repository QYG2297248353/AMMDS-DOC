# Emby

Emby は、デジタルメディアファイルの整理、管理、共有を専門とする強力なマルチメディア管理アプリケーションスイートです。このプラグインを通じて、AMMDS は Emby メディアサーバーとの統合を確立し、メタデータの同期とメディアライブラリ管理を実現できます。

## プラグイン設定

![プラグイン設定](/img/plugin/emby-01.png)

### 基本設定

| パラメータ | 説明 | デフォルト値 |
|------|------|--------|
| 有効状態 | Emby プラグインの有効化を制御 | オフ |
| サービスアドレス | Emby サーバーのネットワークエンドポイント | `http://127.0.0.1:8096` |
| API キー | Emby サーバーの認証トークン | - |
| ユーザー名 | (オプション) Emby サーバーのログインユーザー名 | - |
| パスワード | (オプション) Emby サーバーのログインパスワード | - |
| ユーザーのバインド | バインドする Emby ユーザーアカウントを選択 | - |

![プラグイン設定](/img/plugin/emby-02.png)

### 設定の説明

- **サービスアドレス**：サーバーの実際のネットワークアドレスを使用してください。`localhost` または `127.0.0.1` の使用は避けてください。そうしないと、コンテナ化されたデプロイ環境で接続エラーが発生する可能性があります
- **API キー**：Emby 管理インターフェイスの「システム」→「詳細」→「API キー」で生成できます
- **ユーザーのバインド**：正しいサービスアドレスと API キーを入力して保存した後にのみ、ドロップダウンリストからユーザーを選択できます

### 接続確認

- **ネットワーク接続性**：AMMDS サーバーが Emby サービスアドレスにアクセスできることを確認してください
- **認証情報**：API キーには必要な操作を実行するための十分な権限が必要です
- **サービス状態**：Emby サーバーが実行中で API リクエストに正常に応答できることを確認してください

## 機能特性

- **メタデータの同期**：Emby メディアライブラリとのメタデータの双方向同期をサポート
- **ユーザーのバインド**：特定の Emby ユーザーをバインドして、個別設定を実現
- **状態の監視**：Emby サーバーの実行状態をリアルタイムで監視

## 注意事項

:::warning サービスアドレスの設定
特にコンテナ化されたデプロイ環境では、`localhost` または `127.0.0.1` をサービスアドレスとして使用しないでください。そうしないと、接続エラーが発生する可能性があります。
:::

:::info API キーの権限
システムのセキュリティを確保するために、AMMDS 用に専用の API キーを作成し、適切な権限範囲を付与することを推奨します。
:::
