# 1Panel デプロイガイド

1Panel は、強力なサーバー管理パネルであり、直感的な Web インターフェースを提供し、ユーザーが Linux サーバー内のアプリケーション、Web サイト、ファイル、データベース、大規模言語モデル（LLM）などのリソースを効率的に管理するのを支援します。このガイドでは、1Panel 環境での AMMDS のデプロイ手順について詳しく説明します。

<!-- truncate -->

## サードパーティアプリストアのインストール

この方法は 1Panel v1 および v2 バージョンに適用されます。

### 自動インストール

自動インストール方法では、サードパーティアプリストアのコンテンツが 3 時間ごとに自動的に更新され、常に最新のアプリケーションリソースを取得できるようになります。

#### インストールスクリプト

```bash
curl -sSL https://install.lifebus.top/auto_install.sh | bash
```

#### アンインストールスクリプト

```bash
curl -sSL https://install.lifebus.top/auto_uninstall.sh | bash
```

### 手動インストール

手動インストールスクリプトは、最新のサードパーティアプリストアのコンテンツを一度だけ取得します。その後のストアのコンテンツまたはバージョンの更新には、このスクリプトを再度実行する必要があります。

```bash
curl -sSL https://install.lifebus.top/app_install.sh | bash
```

## ストアのコンテンツを更新

### 操作手順

1. **ストア設定に入る**
   左側のナビゲーションバーからアプリストアに入り、ストア設定ページに進みます。


2. **更新操作を実行**

   ![更新商店内容](/img/deploy/install-1panel-00.png)
   
   #### v1 バージョン
   ストアページの右上隅にある「更新应用列表」ボタンをクリックします。

   ![更新商店内容](/img/deploy/install-1panel-01.png)

   #### v2 バージョン
   ストアページの左上隅にある「同步本地应用」ボタンをクリックします。

## アプリケーションのインストール

### アプリケーションの検索と選択

![安装](/img/deploy/install-1panel-02.png)

1. **AMMDS アプリケーションを検索**
   アプリストアで「ammds」アプリケーションを検索します。

2. **バージョンの選択**
   システムには 2 つのバージョンの AMMDS アプリケーションが表示されます：
   - **互換性バージョン**：ハードウェア構成が低いデバイスに適しています
   - **最新バージョン**：ハードウェア構成が高いデバイスに適しています

   ![安装](/img/deploy/install-1panel-03.png)

### インストールパラメータの設定

インストールボタンをクリックすると、パラメータ設定パネルが表示されます。

#### 基本設定

- **名前**：「ammds」と入力することを推奨しますが、必要に応じてカスタマイズすることもできます
- **バージョン**：最新の機能を取得するには「最新バージョン」を選択することを推奨します
- **ネットワークモード**：特別な要件がない限り、デフォルト設定を保持することを推奨します
- **データ永続化パス**：AMMDS アプリケーションデータを保存するためのサーバー上の絶対パスを入力します
- **WebUI ポート**：8080 を入力することを推奨しますが、必要に応じてカスタマイズすることもできます
- **API ポート**：8081 を入力することを推奨しますが、必要に応じてカスタマイズすることもできます

その他のパラメータはオプション設定であり、実際の要件に応じて入力することができます。

![安装](/img/deploy/install-1panel-04.png)

#### マウントディレクトリの設定

**カスタムマウントディレクトリ**：メディアファイルが存在するディレクトリを入力することを推奨します（オプション）。

![安装](/img/deploy/install-1panel-05.png)

#### ポートアクセスの設定

**「ポート外部アクセス」オプションを必ずチェックしてください**。そうしないと、ネットワークを介して WebUI と API にアクセスすることができません。

「確認」ボタンをクリックして、AMMDS アプリケーションのインストールを開始します。

## インストールの完了

![安装完成](/img/deploy/install-1panel-06.png)

### インストール成功の確認

インストールが完了すると、「已安装」タブでインストール済みの AMMDS アプリケーションを確認することができます。

### 管理機能の紹介

アプリケーションカードには、複数のクイック操作ボタンが提供されています：
- **ターミナル**：アプリケーションコンテナのターミナルにアクセスします
- **ログ**：アプリケーションの実行ログを表示します
- **起動**：アプリケーションサービスを起動します
- **停止**：アプリケーションサービスを停止します
- **再起動**：アプリケーションサービスを再起動します
- **アンインストール**：アプリケーションを削除します
- **再構築**：アプリケーションコンテナを再構築します

## アプリケーションへのアクセス

![访问应用](/img/deploy/install-1panel-07.png)

### アクセス方法

インストールが完了すると、ブラウザを使用して AMMDS の WebUI と API インターフェースにアクセスできます：

- **WebUI アドレス**：http://&lt;サーバーIP&gt;:&lt;WebUIポート&gt;
- **API アドレス**：http://&lt;サーバーIP&gt;:&lt;APIポート&gt;

### デフォルトのログイン情報

- **ユーザー名**：ammds
- **パスワード**：ammds

:::info セキュリティのヒント
初回ログイン後、システムセキュリティを確保するために、デフォルトパスワードをすぐに変更してください。
:::

## 高度な試み

### カスタム設定

Docker コンテナ管理に精通している場合は、1Panel の高度な設定オプションを通じて、AMMDS アプリケーションに対してより細かい設定を行うことができます：

- リソース制限の調整（CPU、メモリ）
- 環境変数の設定
- ネットワークの高度な設定
- ストレージボリュームの管理

### マルチインスタンスデプロイ

異なる機能シナリオを分離する必要があるユーザーは、1Panel で複数の AMMDS インスタンスをデプロイし、異なるポートとデータパスで区別することができます。

## 警告と注意事項

### セキュリティ警告

:::danger 重要なセキュリティのヒント

- **公式ソース**：アプリケーションインストールスクリプトは公式チャネルからのみ取得し、サードパーティのダウンロードソースの使用は避けてください
- **パスワードのセキュリティ**：強力なパスワードを設定し、定期的に変更し、弱いパスワードの使用を避けてください
- **ポートのセキュリティ**：デフォルトポートを変更し、攻撃されるリスクを減らしてください
- **外部アクセス**：実際の要件に応じてポートの外部アクセスを設定し、不要な公開を避けてください

:::

### システム要件

:::info システム構成要件

#### 最小要件

- オペレーティングシステム：Linux ディストリビューション（1Panel インストールをサポート）
- プロセッサ：Intel Core i3 または同等のパフォーマンス
- メモリ：4GB RAM
- ディスク領域：少なくとも 10GB の空き領域
- ネットワーク：ブロードバンドインターネット接続

#### 推奨要件

- オペレーティングシステム：Linux ディストリビューション（1Panel インストールをサポート）
- プロセッサ：Intel Core i5 または同等のパフォーマンス
- メモリ：8GB RAM 以上
- ディスク領域：20GB の空き領域
- ネットワーク：安定したブロードバンドインターネット接続

:::

### 使用上の注意事項

- **データバックアップ**：定期的に AMMDS の設定ファイルと重要なデータをバックアップしてください
- **リソース管理**：サーバーリソースの使用状況を監視し、リソースの過剰使用を避けてください
- **ネットワーク接続**：特にリソースのダウンロードと更新時に、安定したネットワーク接続を確保してください
- **システムメンテナンス**：システムの安定性を確保するために、定期的に 1Panel と関連アプリケーションを更新してください

### トラブルシューティング

#### よくある問題

- **インストール失敗**：ネットワーク接続が正常であるか、ディスク領域が十分であるかを確認してください
- **起動異常**：アプリケーションログを確認して詳細なエラー情報を取得してください
- **アクセス失敗**：ポート設定が正しいか、ファイアウォールがアクセスを許可しているかを確認してください
- **アップグレード失敗**：古いバージョンをアンインストールしてから新しいバージョンをインストールしてみてください

#### 連絡先

解決できない問題が発生した場合は、公式 GitHub リポジトリにアクセスして Issue を提出するか、助けを求めてください。

## ベストプラクティス

### 日常的なメンテナンス

- **定期的な更新**：最新の機能とセキュリティ修正を取得するために、1Panel と AMMDS を最新バージョンに保持してください
- **状態の監視**：定期的にアプリケーションの実行状態とサーバーリソースの使用状況を確認してください
- **キャッシュのクリーンアップ**：アプリケーションキャッシュを定期的にクリーンアップし、ストレージスペースの使用を最適化してください

### 設定の最適化

- **リソース割り当て**：サーバーのパフォーマンスに応じて、アプリケーションのリソース制限を合理的に割り当ててください
- **ネットワークの最適化**：アプリケーションアクセスのスムーズさを確保するために、合理的なネットワーク設定を構成してください
- **ストレージ計画**：管理とバックアップを容易にするために、データストレージパスを合理的に計画してください

### セキュリティの強化

- **定期的なパスワード変更**：定期的にログインパスワードとアクセス認証情報を更新してください
- **アクセス制御**：実際の要件に応じてアクセス制御ポリシーを設定してください
- **ログ監査**：定期的にアプリケーションログを確認し、異常な状態を及时に発見してください

### コミュニティへの参加

- **問題のフィードバック**：製品の改善を助けるために、問題が発生したら及时にフィードバックしてください
- **経験の共有**：コミュニティの議論に参加し、使用経験とベストプラクティスを共有してください
- **コードの貢献**：能力がある場合は、プロジェクトの開発に参加し、コードを貢献してください
