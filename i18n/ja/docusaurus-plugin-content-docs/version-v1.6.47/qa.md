# よくある質問

使用中によくある質問です。

<!-- truncate -->

## 🚀 デプロイに関する質問

<details>
  <summary>🔍 ログに CPU が x86-64-v2 をサポートしていないと表示されますか？</summary>
  
  お使いのデバイスの CPU が古すぎます。`ol8` タグのついたイメージを選択してください。
</details>

<details>
  <summary>⌛ 初回デプロイ後に応答がないのはなぜですか？</summary>
  
  初回デプロイ時、システムは必要なリソースファイルをダウンロードして初期化する必要があります。これには時間がかかります。数分待ってください。まだ応答がない場合は、ページを更新してみてください。
</details>

<details>
  <summary>⏬ リソースのダウンロードが非常に遅いですか？</summary>
  
  リソースファイルが大きいため、ダウンロード速度が遅くなる可能性があります。以下の方法で速度を上げることができます：
  - ネットワーク環境を変更する。たとえば、HTTP_PROXY および HTTPS_PROXY 環境変数を設定してネットワークを高速化する
  - リソースを手動でダウンロードしてアプリケーションを再起動する
</details>

<details>
  <summary>📁 監視ディレクトリの登録に失敗し、起動後すぐに終了しますか？</summary>
  
  これは通常、設定ファイルまたは環境変数が正しくないことが原因です。アプリケーションの設定ファイルが正しいか確認し、環境変数が正しく設定されていることを確認してください。
  
  以下のログが表示される場合：
  ```
  Error creating bean with name 'directoryWatcherService': Instantiation of supplied bean failed
  User limit of inotify instances reached or too many open files
  User limit of inotify watches reached
  ```
  
  システムの inotify インスタンス制限を増やしてみてください：
  ```bash
  echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
  echo fs.inotify.max_user_instances=524288 | sudo tee -a /etc/sysctl.conf
  sudo sysctl -p
  ```
</details>

## 💡 使用に関する質問

<details>
  <summary>🖱️ メニューをクリックしても応答がないのはなぜですか？</summary>
  
  これは通常、ページが一時的にフリーズしているか、リソースが完全に読み込まれていないためです。ページを更新してみてください。問題が解決しない場合は、ブラウザコンソールにエラーメッセージが表示されていないか確認してください。
</details>

<details>
  <summary>⚪ 開いた後に白い画面になるのはなぜですか？</summary>
  
  これは通常、ネットワークリソースが完全に読み込まれていないためです。ページを更新してみてください。問題が解決しない場合は、ブラウザコンソールにエラーメッセージが表示されていないか確認してください。ネットワーク接続を安定させ、ネットワークの乱れを避けることをお勧めします。
</details>

<details>
  <summary>☁️ クラウドストレージの .strm ファイルをマウントするにはどうすればよいですか？</summary>
  
  クラウドストレージのユーザーは、まず Alist、alist-strm、NetMount などのツールを使用してクラウドストレージのファイルをローカルに .strm ファイルとしてマウントし、それをアプリケーションにマウントすることができます。
  
  スキャン設定で、メディア識別タイプを `strm` に設定してローカルの .strm ファイルをスキャンします。
</details>

<details>
  <summary>🌍 タイムゾーンエラー、システム時間が正しくないですか？</summary>
  
  これは通常、アプリケーションのタイムゾーンとサーバーのタイムゾーンが一致していないためです。アプリケーションのタイムゾーン設定が正しいか確認し、サーバーのタイムゾーンと一致していることを確認してください。
  
  - TZ 環境変数を設定している場合は、正しいタイムゾーンに設定してください
  - ネットワークプロキシを使用している場合は、プロキシサーバーのタイムゾーンがサーバーのタイムゾーンと一致していることを確認してください
  - Docker を使用してデプロイしている場合は、Docker コンテナのタイムゾーン設定が正しいことを確認してください
  
  自動タイムゾーン検出を実行するには、インターネットにアクセスできる必要があることに注意してください。
  
  プロキシネットワークの場合は、suning.com、taobao.com、meituan.com などの国内プロキシドメインの使用を避けてください。
</details>

<details>
  <summary>🔌 [Metatube] 404 (見つかりません)</summary>
  
  MetaTube プラグインのサーバーアドレスが正しく設定されているか確認してください。カスタムドメインを使用している場合は、ドメイン解決が正しいことを確認してください。リバースプロキシを使用している場合は、リバースプロキシの設定が正しいことを確認してください。MetaTube コンテナのログが正常であるか確認し、エラーメッセージがないことを確認してください。
  
  [MetaTube プラグイン設定ドキュメント](https://ammds.lifebus.top/guide/plugins/metatube/) を参照してください。
</details>
