# R18Dev

R18Dev 是一个源于 R18.com 关闭后的数据保存项目，旨在保留和提供 JAV 影片的元数据信息。通过 R18Dev 插件，AMMDS 可以自动获取影片的标题、演员、发行日期、片商、类别等元数据，帮助你快速构建完整的媒体库信息。

## 插件配置

![基本配置](/img/plugin/r18dev-01.png)

### 基本配置

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 启用状态 | 控制是否启用 R18Dev 插件 | 关闭 |
| 服务地址 | R18Dev 服务的访问地址 | `https://r18.dev` |
| 影视检索 | 控制是否开启影视元数据检索功能 | 关闭 |
| 在线检索 | 开启后优先通过网络检索，而非使用本地导入的数据 | 关闭 |

### 配置说明

- **服务地址**：默认使用官方地址 `https://r18.dev`，如果部署环境网络受限，可以配置代理地址或自建镜像
- **在线检索**：开启后，AMMDS 在刮削影片时会优先通过网络实时检索元数据；关闭时则仅使用本地已导入的数据库进行匹配

:::info[数据来源]
R18.dev 数据库基于原 R18.com 的数据构建，并持续从多个来源更新，提供较为完整的 JAV 影片元数据覆盖。
:::

## 数据管理

![数据管理](/img/plugin/r18dev-02.png)

### 数据统计

数据统计区域展示当前已导入的数据概览，包括：
- **影片数量**：已导入的影片总数
- **演员数量**：已导入的演员信息总数
- **导演数量**：已导入的导演信息总数

通过这些统计数据，你可以快速了解本地数据库的覆盖情况。

### 数据导入

支持通过以下方式导入数据：
- **选择文件**：点击文件选择区域，手动选择数据库文件进行导入
- **拖入文件**：将数据库文件直接拖拽到导入区域

模块右上角提供快捷操作按钮：

- **下载页面**（`https://r18.dev/dumps`）：打开数据库文件下载页面，用户可以自行选择历史版本的数据库文件进行下载
- **获取最新数据包**（`https://r18.dev/dumps/latest`）：点击后自动触发最新数据包的下载，下载完成后需要手动导入到系统中

:::tip[导入建议]
建议定期获取并导入最新数据包，以保证元数据检索的准确性和覆盖范围。
:::

## 测试模块

测试模块提供以下功能，用来验证插件配置和服务是否正常：

![检索测试与连接测试](/img/plugin/r18dev-03.png)

| 功能 | 说明 | 作用 |
|------|------|------|
| 检索测试 | 输入影片ID、演员名称或导演名称等关键词 | 验证元数据检索功能是否正常 |
| 连接测试 | 点击测试按钮 | 验证与 R18Dev 服务的连接是否成功 |

:::info[测试说明]
- 连接测试只验证网络是否连通，不保证元数据检索一定成功
- 检索测试建议使用标准的 JAV 番号格式（如 `SSIS-001`、`ABP-999`），这样匹配准确率更高
:::

## 注意事项

:::tip[使用建议]
编码格式推荐使用标准的 JAV 番号格式（如 `SSIS-001`、`ABP-999`），这样匹配准确率更高。
:::

:::warning[地域访问]
R18Dev 服务可能对部分地区的访问有限制，如果遇到连接问题，建议检查网络环境或配置代理访问。
:::

:::note[服务状态]
R18Dev 仍处于持续开发阶段，数据可能存在部分遗漏或错误。如遇到数据问题，可以关注项目更新或通过官方渠道反馈。
:::

## 常见问题

### 连接服务失败

**可能原因**：
- 网络连接问题
- 服务地址配置错误
- 地域访问限制

**解决方案**：
- 检查网络连接是否正常
- 确认服务地址格式是否正确
- 尝试使用代理或镜像地址

### 检索不到元数据

**可能原因**：
- 影片编号格式不正确
- R18Dev 数据库中暂无该影片信息
- 网络环境受限导致检索超时

**解决方案**：
- 检查影片编号格式是否正确
- 换用更常见的编号格式重试
- 确认网络环境能正常访问 R18Dev
- 尝试导入最新数据包后重试

### 数据导入失败

**可能原因**：
- 数据库文件格式不正确
- 导入的数据包版本与当前系统不兼容
- 文件损坏或不完整

**解决方案**：
- 确认下载的数据库文件完整无损
- 尝试使用其他历史版本的数据库文件
- 重新下载最新数据包后再次导入
